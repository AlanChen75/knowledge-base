學點基礎才不會搞砸
那些說「完全不用懂程式碼」的人是在騙你。
你不需要會寫程式，但要懂你家網站用的技術框架的基本概念。AI 傾向找「最簡單的路徑」解決問題，但最簡單不等於最好，有時候反而會惹麻煩。

我建議至少了解：
→ HTML/CSS/JS 是什麼（網頁的骨架、外觀、互動行為）
→ 你家網站用的框架基礎（例如 Next.js）
→ SEO 原理（Meta tags 是什麼、Sitemap 是什麼）
→ 部署是什麼意思（把東西放上網路讓別人看到的過程）
不懂的時候叫 Claude：「用解釋給五歲小孩聽的方式（ELI5）告訴我你做了什麼。」

請工程師幫你把關
涉及產品或面向用戶的改動，發布前傳訊息問工程師：「這樣改合理嗎？」

我曾經為了「效能優化」把手機版畫面搞壞，好險是在測試環境（staging，一個跟正式網站一樣但用戶看不到的測試空間）發現的。這一步不能省。

保持人在迴路裡（Human-in-the-loop）
不要試圖全自動化。行銷需要個性，AI 沒有個性。
讓 AI 做苦差事（草稿、數據整理），你負責加進去：品牌語氣、策略決策、編輯判斷、創意方向。

▍ 缺點和風險（不只說好的）
→ 不一定比較快：熟練的工程師絕對比我快。重點是獨立性，不需要等別人排程。
→ 你會犯錯：AI 生成不代表正確，一定要測試、檢查。
→ 有學習曲線：第一週很挫折，可能把開發環境搞壞，這很正常。
→ 不能取代策略：Claude Code 是執行者，策略、定位、創意還是在你腦袋裡。

▍ 常見問題
Q: 一定要用終端機（那個黑色視窗）嗎？
A: 不用！用 Claude Desktop App 就好，介面像聊天軟體，還支援拖拉檔案。我自己也是用這個。

Q: 我需要會寫程式嗎？
A: 不用精通，但懂基礎會有幫助。知道 Next.js 基礎可以避免把網站搞亂。不懂的時候用 ELI5 指令讓它解釋給你聽。

Q: Claude Code 能取代開發者嗎？
A: 不能，這也不是目的。它讓行銷人能獨立處理活動頁面、簡單自動化和日常優化。複雜的架構還是需要工程師。

Q: 需要接哪些工具？
A: 我個人用的：
→ Ahrefs：SEO 數據與文章流量監測
→ Firecrawl：抓取網頁數據
→ Google Search Console：搜尋成效數據
→ Figma：設計稿
→ Hex：數據分析
→ Composio：用來串接其他行銷工具的橋樑（就像一個讓不同工具互相溝通的翻譯官）
