▍ 什麼是 Claude Code？
簡單說：Claude Code 就是住在你電腦終端機（terminal，就是那個全黑的指令輸入視窗）裡的 Claude。

它是一個工具，可以讀取你整個專案的所有檔案、跨多個文件進行修改、執行指令。就像你的團隊多了一個工程師，只是你用說話的方式下指令。

跟網頁版 Claude 最大的差別在於：你不用來回複製貼上程式碼，Claude Code 直接「住在你的專案裡」，看得到整個目錄，理解前後文，直接動手改。你甚至可以讓它存取你的本機檔案，幫你整理桌面截圖之類的事情。

━━━━━━━━━━━━━━━━━━━━
▍ 怎麼開始用
安裝方式（超簡單一行指令）：
curl -fsSL https:/ claude.ai /install.sh | bash
進到你的專案資料夾後輸入 claude 就啟動了。
像跟開發者聊天一樣跟它對話就好。
如果你怕終端機： 可以直接用 Claude Desktop App（我自己也是這樣用）。介面就像一般聊天軟體，可以拖拉截圖、貼上圖片，完全不需要面對黑色視窗。

▍ 行銷人的 Claude Code 實戰應用
━━━━━━━━━━━━━━━━━━━━
1. 打造客製化追蹤工具（Custom Agents）
我已經不再幻想「如果有個工具能解決這個問題就好了」- 現在我都直接花不到一小時自己做一個。

Firecrawl 提及追蹤器
這是一個簡單但超好用的自動工具，每週一我會收到一份報告，整理過去一週 Reddit、Hacker News、Twitter 或小眾論壇上關於我們的討論。

為什麼這很有用？
當我看到真實用戶分享案例或稱讚特定功能，我就把它們寫進當週的內容裡。引用真人的回饋比自己老王賣瓜更有說服力。我也用這個工具追蹤「網頁抓取」「搜尋 API」等廣泛議題，看看開發者這週在問什麼，下週的內容題材就有了。

━━━━━━━━━━━━━━━━━━━━
2. 自動更新流量下滑的文章（Content Decay）
這個設定稍微複雜，但每週幫我省了好幾個小時。

流程是這樣的：
→ Ahrefs（SEO 工具）監測流量數據
→ 排程工作（Cron job，就是讓電腦定時自動執行某件事）每週找出流量下滑前 10 的文章
→ Slack 通知推給我
→ 我在 Slack 上挑出 3 篇高潛力文章，告訴 Claude 要怎麼優化
→ Claude 負責苦差事：更新舊數據、刷新程式碼範例、增加新段落、修復內部連結

重要：我不會讓它自動發布。每一項改動我都親自審閱。全自動生成的內容通常很普通，也不符合品牌語氣，這關我自己過。

━━━━━━━━━━━━━━━━━━━━
3. 內連優化工具（省下 $30 美元/月）
以前我付費買 SaaS 工具來找內部連結機會（就是在文章裡加上連回自家其他頁面的連結），但給的建議都很普通。

因為 Claude Code 能讀取我們整個網站的所有檔案，它能根據語意（真的理解內容在說什麼）而不是關鍵字來建議連結位置。它知道哪些錨點文字（anchor text，就是超連結顯示的那段字）更自然，哪些頁面需要被連到，而且不會看起來像垃圾連結。
效果比付費工具好，而且免費。

━━━━━━━━━━━━━━━━━━━━
4. 利用 Firecrawl 抓取即時網路數據
在 Firecrawl 我們當然把自家工具整合進 Claude Code 了（不然就尷尬了）。這讓 Firecrawl 成為我 AI 工作流中的完整數據層。
它現在是我的研究助手、趨勢分析工具、寫作輔助、事實查核助手。

舉個例子：寫競品比較文時，我直接下指令：
「用 Firecrawl 抓這 5 家競品的定價頁面，做成一張比較表。」
它負責那些討厭的抓資料工作，我專心分析和寫作。

━━━━━━━━━━━━━━━━━━━━
5. 挖掘用戶真實問題（FAQ 生成）
我用 /agent 指令（一個呼叫 AI 代理模式的指令）幫每篇文章找 FAQ。

我的提示詞（Prompt）長這樣：
「用 Firecrawl 找出過去 3 個月內，[目標受眾] 在 [他們聚集的地方] 問了哪些關於 [主題] 的問題。」這讓我能回答人們真正想知道的事，而不是瞎猜他們會搜尋什麼。

━━━━━━━━━━━━━━━━━━━━
6. SEO 技術雜務自動化
圖片優化
以前搞圖片格式、壓縮、調整尺寸真的超煩。現在全自動：
→ 轉成 WebP 格式（省 30-40% 檔案大小）
→ 調整成正確尺寸（避免版面位移，這是一個影響 SEO 評分的技術問題）
→ 自動加上 Alt text（圖片替代說明文字，對搜尋引擎和無障礙功能很重要）

網站健康度審核
上週我讓 Claude Code 幫我做了一次審核，它丟出 20 幾個建議，包括 Lazy loading（讓圖片延遲載入、節省頁面讀取時間）和 JS 優化這類技術項目。
我快速跟工程師確認不會把網站搞掛之後，就直接逐一修掉了。

━━━━━━━━━━━━━━━━━━━━
7. 追蹤 AI 能見度（我的 AEO 實驗）
AEO（Answer Engine Optimization，答案引擎優化）是試圖讓你的內容更容易被 ChatGPT、Perplexity 等 AI 工具引用和推薦的做法。
我的做法：

從 Vercel（我們的網站部署平台）拉出 Log（系統紀錄），看哪些 AI 機器人（像 ChatGPT-User 這類爬蟲）爬了哪些頁面
把數據餵給 Claude，問它：「用戶可能是問了什麼問題，才讓這些機器人來爬這些頁面？」
Claude 根據網站內容給出建議，我再用 Profound 這個工具來追蹤我們的 AI 能見度

說真的，現在還有很大程度是在猜測，但至少我是基於真實的爬蟲數據在猜。

━━━━━━━━━━━━━━━━━━━━
8. 很亂但很有效的筆記系統
我把所有看到的有趣數據、論文、推文連結都丟進一個本機文字檔。

寫文章前，我叫 Claude：「去掃描那個檔案，看有沒有跟這次主題相關的素材，有的話自然地放進文章裡。」就像有個記憶力超強的研究助理，永遠不會忘記你三個月前看過的那個關鍵數據。

▍ 給行銷人的入門建議（真心話版）
把重複的事變成固定用法（Skills）
如果你覺得某件事會做第二次，就把它存成 Claude 的 Skill（可以理解成：你幫 Claude 寫好一份工作 SOP，下次直接叫它照這份跑）。

這對維持寫作風格特別有用。我有一個叫 hiba-writing-style 的 Skill 檔，每次讓 Claude 幫我寫東西，它會自動套用我的語氣。

做完任務後叫 Claude：「把這個流程和預期產出文件化，變成一個 Skill。」它會自己整理。
