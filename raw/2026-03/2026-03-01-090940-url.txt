大家好，我是阿紫BigBig，萬法歸一 (WFGY作者)  github 1.5k

最近很多人在研究 openClaw，龍蝦本身其實就是一個強大的 agent 系統。只要是工程師，前期可能先拿來做幾個小任務測試，結果用著用著，就會開始一路往下接工具、資料來源、工作流程，最後自然就變成一整條 AI pipeline。問題是，當你把 RAG、向量庫、各種工具全都塞進同一個 agent 裡，只要回覆開始亂飄，常常沒人講得清楚到底是哪一層出事，大家只會感覺「好像哪裡怪怪的」。

我這邊有一份自己整理出來的 RAG「16 問題清單」，本質上就是一個「語義防火牆 semantic firewall」。可以把它想成是在任何 RAG 或 agent 系統外面，先架一圈檢查點。你在 openClaw 裡設計流程之前，或者已經把 pipeline 串好但開始出現幻覺時，就可以用這 16 格去逐格對照，從資料 ingest、chunk 切法、embedding 設定、vectorstore 參數，到 prompt 設計、工具路由、評估方式，都有對應的錯誤型態可以檢查，不會只能憑感覺亂猜。

這份清單目前已經被幾個學術與開源生態收錄，哈佛 MIMS Lab 的 ToolUniverse、卡達計算研究院 QCRI LLM Lab 的 Multimodal RAG Survey，以及一些 AI / DataScience 的大型 awesome list，都把它當成 RAG 除錯與壓力測試的參考表，都把它當成 RAG 除錯和壓力測試的參考表。意思是，已經有不少團隊實際拿去用，覺得真的有幫助，才願意放進他們的官方整理裡面，而不是我自己單方面說它很厲害。

清單在這裡，頁面上就是主表，往下拉還有補充說明，中文註解也都在同一頁：

https://github.com/onestardao/WFGY/blob/main/ProblemMap/README.md

只要你有在做下面這幾種事情，其實都用得上

自己搭 RAG 知識庫、做公司內部搜尋助手、用 openClaw 或其他 agent framework 串各種工具、幫客戶做文件助理或客服 bot，只要結果開始出現奇怪的幻覺、答非所問、資訊遺漏，又說不出源頭在哪裡，就可以直接打開這 16 格，對照看是落在哪一類錯誤，然後沿著那一格的方向去修。

整份是 MIT 授權，只有文字清單，不綁特定框架，也不限定模型。你可以拿去公司內部當 checklist 用，或者改成自己的版本放進專案文件裡，都沒有問題。

順便講一下，這份清單只是我整個專案 WFGY（萬法歸一）裡面的一小塊。WFGY 本身是在做底層推理引擎和語義診斷的，裡面還有 Drunk Transformer、張力宇宙、各種數學模組和實驗，如果好奇可以自己進 repo 慢慢挖。不過今天先不一次全部打開，只是把這份 RAG 16 問題清單先丟出來。
