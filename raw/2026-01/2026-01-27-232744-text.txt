這篇貼文來自 AI 大神 Andrej Karpathy，他曾是 OpenAI 的創始成員並長期主導特斯拉的 AI 研發。這篇筆記詳盡記錄了他最近幾週高強度使用 AI 寫程式的深刻轉變，特別是他如何從一位資深工程師，在短短一個月內進化成以 AI 為主力的開發模式。

他在貼文裡描述一種「相位轉移」式的變革，指出當前的 AI 能力已經跨越了某種門檻，讓程式開發從以往的一行行手動輸入，轉變為由人類下達高層次指令、AI 負責大範圍執行的全新樣貌，以下是他的貼文內容。

"程式開發工作流

過去幾週頻繁使用 Claude 撰寫程式，有一些隨筆感觸。隨着大型語言模型（LLM）編碼能力的提升，我與許多人一樣，工作流程在短時間內發生了劇變。

十一月時，我大約是 80% 手動加自動補完、20% 使用 Agent；到了十二月，已演變為 80% 由 Agent 編寫、20% 進行人工修改與潤飾。換言之，我現在大多是在用英文編寫程式，甚至帶點羞愧地用文字「吩咐」模型該寫什麼。

雖然這有點打擊自尊，但能以一口氣完成大區塊的程式碼更動，來操作軟體的威力實在太強大了。一旦你適應、配置並學會如何運用，並搞清楚它的能力邊界，這種轉變就變得極具價值。這是我二十年程式生涯中，基本開發流程發生的最大變革，且僅在短短數週內完成。我預期有超過雙位數百分比的工程師正經歷類似轉變，然而一般大眾對此的覺察可能還不到個位數。

IDE、Agent 群體與可靠性

我認為目前「不再需要 IDE」或「Agent 群體」的過度吹捧都言過其實。模型絕對會犯錯，若你面對的是真正重要的程式碼，建議還是在旁邊開一個寬大的 IDE 視窗，像鷹隼般盯著它看。

現在的錯誤類型已有所改變，不再是單純的語法錯誤，而是類似粗心、倉促的初級工程師會犯的那種隱晦觀念錯誤。最常見的情況是模型替你預設了錯誤的前提，且在未經確認的情況下晉自執行。

此外，它們不會管理自己的困惑，不會主動尋求澄清，也不會揭露不一致之處或提出權衡方案。在該拒絕時，它們不會反駁，且依然過於溫順。雖然「計畫模式」有所改善，但仍需更輕量化的行內計畫功能。

模型也傾向於將代碼與 API 複雜化，導致抽象層冗贅，且不會自動清理廢棄代碼。它們可能會花上一千行去實現一個低效、臃腫且脆弱的結構，直到你提醒「難道不能簡單點嗎？」，它們才會立刻改寫成精簡的一百行。

即便在 CLAUDE .md 寫入指令，它們有時仍會因為副作用而改動或刪除原本不相關的註解與代碼。儘管問題不少，但這依然是巨大的淨提升，很難想像回到全手動編寫的時代。簡言之，每個人都有自己的步調，我目前的配置是左側開啟幾個 Ghostty 視窗執行開發對話，右側則用 IDE 檢視代碼並進行手動修改。

韌性

觀察 Agent 不懈地解決問題非常有趣。它們不知疲倦、不覺沮喪，在人類可能早已放棄、改日再戰的地方，它們會持續嘗試。看著它與難題纏鬥許久，最終在三十分鐘後勝出，那種「感受到通用人工智慧（AGI）」的瞬間令人震撼。你會意識到「耐力」曾是工作的核心瓶頸，而有了 LLM 後，這個上限已被大幅拉高。

產能加速

要衡量 LLM 帶來的「加速」並不容易。我確實感覺處理既定任務的速度變快了，但主要影響在於我「做得更多了」。原因有二：首先，我可以開發許多以前認為不值得動手的東西；其次，我可以涉足以前受限於知識或技能而無法處理的領域。因此，這與其說是加速，不如說是一種能力的擴張。

槓桿作用

LLM 極擅長重複執行直到達成特定目標，這也是「感受 AGI 魔力」的關鍵所在。與其告訴它什麼做，不如給它成功準則並放手讓它執行。讓它先寫測試再通過測試，或是搭配瀏覽器 MCP 工具使用。先寫出邏輯正確的簡易演算法，再要求它在保持正確性的前提下進行優化。將思維從命令式轉向宣告式，能讓 Agent 持續發揮槓桿作用。

樂趣

我沒預料到有了 Agent 後，寫程式會變得更有趣。因為大量填空式的瑣事消失了，留下的多是創意部分。我不再那麼容易感到卡關，且更有勇氣嘗試，因為總能與它攜手取得進展。不過我也聽過相反的觀點，LLM 編碼可能會將工程師分化為兩類：一類是純粹熱愛「寫程式碼」的人，另一類則是熱愛「打造產品」的人。

能力萎縮

我發現自己手寫程式的能力正逐漸萎縮。大腦中「生成程式碼」與「辨識程式碼」是不同的功能。由於程式設計涉及大量語法細節，即便手寫變得生疏，你依然能順利地審閱代碼。

AI 廢材末日

我正嚴陣以待 2026 年「AI 廢材末日」的到來，屆時 GitHub、Substack、arXiv 與社群媒體將充斥著 AI 生成內容。我們也會看到更多打著 AI 旗號的「生產力表演」，與真正的進步並存。

思考與結論

我有幾個疑問。首先是「十倍工程師」會如何演變，頂尖與平均工程師的生產力差距可能會大幅拉開。其次，在具備 LLM 武裝後，通才是否會全面超越專才，畢竟模型在微觀填空上優於宏觀策略。此外，未來的 LLM 開發體驗會像什麼，是像玩《星海爭霸》、《異星工廠》，還是像在演奏音樂？最後，社會中有多少比例仍受困於數位知識工作的瓶頸？

這一切將我們帶向何方？LLM 的 Agent 能力（特別是 Claude 與 Codex）在 2025 年十二月左右跨越了某種連貫性的門檻，引發了軟體工程及相關領域的相位轉移。目前「智能」部分顯然超前於工具整合、組織工作流與知識普及。隨著產業開始消化這些新能力，2026 年將會是充滿動能的一年。"
