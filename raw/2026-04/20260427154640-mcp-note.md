# Meeting Capture Studio Phase 2B 升級紀錄

## 📌 摘要

Phase 2B 升級兩件事：(1) v4 Term Discovery 取代 L1 chunk-based 校正，加 post-validate hard rule 解決 LLM 高頻詞綁架/自造詞災難；(2) Stage 3 摘要從單一 prompt 升級為 YAML 5-stage 架構（A vs C 實驗驗證 C 完勝），整體跑全免費 Gemini Free 主 + OpenRouter 備援。

## 🔑 關鍵要點

1. **v4 Term Discovery 一次性 LLM 三階段思考**：主題識別 → 高頻詞分類（A 合理術語/B 錯字/C 中性詞）→ 為錯字找正確版，post-validate 三條 hard rule（拼音編輯距離 ≤ 2、長度差 ≤ 1、correct 必須在 raw 或 yaml）

2. **三層 self-learning 機制**：Layer 1 LLM 預建領域 glossary（120 個能源/儲能術語）→ Layer 2 v4 在每場挖新 cluster → Layer 3 人工 review 後補進 `terms/*.yaml` wrong list，下次同主題會議直接 deterministic 修

3. **YAML 5-stage 摘要架構**取代單一 prompt：Stage 1 議題綱要 + Stage 2×N 逐議題詳細 + Stage 3a/3b/3c 決議/未解/行動項目，prompt 鎖死字數/句型/層級規範

4. **A vs C 實驗驗證**（Gemini Free Tier）：A (2-call) 議題深度 285 字、決議/行動全 SKIP；C (11-call) 議題深度 445 字（+50%）、決議 4 條 + 行動 11 條齊備。**選 C** 上 production

5. **全免費三層 fallback**：Gemini 2.5 Flash 主（15 RPM / 1500 RPD，136 場/天容量）+ OR Ling 1T:free + OR Llama 70B:free，429/503/402 自動 alternate

## 💬 金句摘錄

> 「LLM 不會主動修錯字，會直接用 raw 中的字當議題名」— 摘要必須在 raw 完全清乾淨後才跑

> 「raw 高頻詞白名單會包含錯字（除能 12 次、靜靈 6 次）」— 純頻率白名單路徑有結構性缺陷，必須加 LLM 二次過濾

> 「單次 Stage B 同時要寫議題+決議+未解+行動，LLM 集中寫議題後對 3 區塊 SKIP」— 為什麼 A 失敗、C 必須分階段

## 🧠 概念連結

- **Pinyin Regularization for Chinese ASR**：character error rate 5-30%、pinyin error rate 1.5-20%，拼音作為 LLM prompt 輔助線索能大幅減 hallucination（Tang et al. 2024）
- **Whisper Courtside Edition multi-agent**：Topic Classification + Jargon Extraction + Decision Filter 架構，跟 v4 一致
- **ChFT full-text correction**：JSON 輸出 vs plain text 差異巨大（22.4% vs 失敗）；segment 校正勝於 article 校正
- **Notion 摘要逆向工程**：粗體小標 + 冒號 + 簡述、子項目用具體事物名（產品/工具/數字/人名）、議題標題 6-10 字名詞片語

## 💡 與我的連結

這個架構驗證了「自學習迴路」是必要的：第一場會議用 v4 LLM 挖 cluster → 人工 review 補 yaml → 下一場同主題會議從 Layer 0 yaml 字典直接修（不靠 LLM）→ 累積後新場 LLM 補位率下降，成本與品質雙贏。

對未來教學模組的意義：開源後學員 fork，他們的 yaml 累積就是他們自己的領域知識資產。

## 📝 我的註解與思考

兩個 prompt leak 教訓重要：
1. v4 leak 版直接把 ground truth cluster 列為「範例」→ 100% 修復率是抄答案
2. yaml summary leak 版用 Notion 摘要的具體子議題名當「範例」→ 5945 chars 是被引導

清版（範例改成跨領域示範）後性能下降明顯，但這才是模型真實能力。Production 上要避免任何「特定場景」的範例污染 prompt。

A vs C 對比是教科書級的「分階段強制聚焦」價值證明。LLM 一次任務太多會偷懶（A 對 3 區塊全 SKIP），分階段聚焦才能展現完整能力（C 抓到 4 條決議 + 11 條行動項目）。
