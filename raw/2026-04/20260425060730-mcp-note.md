# Pascal Editor 開源 3D 建築編輯器

## 📌 摘要
Pascal Editor 是一款完全在瀏覽器內運行的開源 3D 建築編輯器,採用 React 19 + Next.js 16 + WebGPU 技術堆疊,GitHub 累積 13,100+ stars,以 MIT 授權釋出。它示範了現代 Web 技術如何把過去必須仰賴 AutoCAD/Revit 等高價 native 軟體的 3D 建模工作,改造成「打開網頁就能用」的零門檻體驗,並融入 AI 輔助設計與 GitHub-style Fork 文化。

## 🔑 關鍵要點

1. **破壞式定價結構**:AutoCAD 標準版年費約 2,680 美元、Revit 約 3,185 美元,Pascal Editor 完全免費、開源、免安裝、免註冊
2. **全瀏覽器執行**:採用 React Three Fiber + WebGPU,GPU 直接參與渲染,效能接近桌面 native 軟體
3. **本地優先資料儲存**:使用 IndexedDB 儲存所有資料,無需伺服器、無需帳號,隱私友善
4. **AI 輔助起稿**:輸入「有泳池的別墅」這類自然語言即可生成 3D 模型初稿,再人工調整
5. **GitHub-style 社群文化**:支援 Fork、按讚、瀏覽次數,讓建築設計可以像程式碼一樣版本管理
6. **完整建築建模支援**:牆面、地板、天花板、屋頂、門窗、家具,支援多樓層管理 (堆疊/爆炸/單層獨顯三種視圖)

## 🧱 技術架構深度解析

### 整體技術堆疊
- **前端框架**:React 19 + Next.js 16
- **3D 渲染**:Three.js 的 WebGPU renderer + React Three Fiber
- **狀態管理**:Zustand + Zundo 中介層 (支援最多 50 步 undo/redo)
- **幾何運算**:three-bvh-csg 處理布林運算
- **Monorepo 工具**:Turborepo + Bun (套件管理器)
- **資料儲存**:IndexedDB (瀏覽器本地)

### 三層套件拆分 (harness 式架構)
- **`@pascal-app/core`**:節點結構、場景狀態管理、幾何生成邏輯
- **`@pascal-app/viewer`**:3D 渲染、相機控制、後製效果
- **`apps/editor`**:UI 元件與編輯工具

### 三個獨立 Zustand store (關注點分離)
- **`useScene`**:場景資料 + CRUD 操作
- **`useViewer`**:選取狀態 + 顯示模式
- **`useEditor`**:工具切換 + 面板偏好

### 資料模型設計
- **階層式組織**:Site → Building → Level → {Walls, Slabs, Ceilings, Roofs, Zones, Scans, Guides}
- **扁平字典儲存**:不用巢狀樹狀結構,而是用扁平字典存所有節點 → 更容易做 diff、版本管理、序列化
- **Registry pattern**:各類元素 (牆、樓板、家具) 透過獨立 System 處理幾何生成
- **Dirty flag 機制**:節點變動時才標記為 dirty,下一幀重算 → 避免不必要的渲染

## 🧠 概念連結

- **Harness Engineering**:Pascal Editor 的三層套件拆分 + 三個 store 分離,本質就是一套 agent harness 的設計模式 — 資料層、呈現層、互動層各司其職。對 Claude Code Skill/Subagent 架構設計有直接借鑑價值
- **Modular MVP Infrastructure**:這套 stack (Bun + Turborepo + IndexedDB + WebGPU) 特別適合「前端重、後端輕」的 MVP 原型,呼應個人四個網站的 modular 哲學
- **AI-as-Scaffolding**:把 LLM 定位成「設計起點」而非「設計終點」,使用者再做細修 — 這是個比「AI 全自動生成」更務實、更可控的產品定位
- **Fork Culture 在非程式領域的擴散**:把 GitHub 的版本管理 + 社群協作模式套用到建築設計,等同於把「software thinking」滲透到 AEC (Architecture, Engineering, Construction) 行業
- **WebGPU 普及里程碑**:作為新世代 Web 圖形標準,Pascal Editor 是少數真正把 WebGPU 用到實用場景的專案

## 💡 與我的連結

### 1. Claude Code Skill/Subagent 架構參考
Pascal Editor 的套件拆分 (core/viewer/editor) + 三 store 分離,可以直接對映到 agent 系統設計:
- `core` ≈ Subagent 的 reasoning/state layer
- `viewer` ≈ 對使用者的呈現層 (CLI 輸出、報告生成)
- `editor` ≈ 互動式控制層 (Commands、Skill 觸發)

### 2. NILM 研究可視化潛力
HTF-CNN 未來如果要做設備分解結果的視覺化展示 (例如能耗流的 3D 可視化、工廠 layout + 設備識別疊加),React Three Fiber + WebGPU 比 Matplotlib/Plotly/Dash 能做出更有說服力的學術 demo,也適合做成期刊論文的補充材料 (supplementary)

### 3. 課程素材 (AI 智慧製造 + AI 永續課程)
這是極佳的「開源 + AI + Web 顛覆傳統專業軟體」案例,可用來講授:
- **技術民主化趨勢**:CAD/BIM 的 SaaS 與開源化
- **WebGPU 能做到什麼**:破除「重運算必須 native」的迷思
- **AI 在專業工具中的定位**:scaffolding vs. automation 的設計哲學差異

### 4. MVP infrastructure 啟發
Bun + Turborepo + IndexedDB 的組合,適合考慮應用在未來的個人/業務網站新模組,特別是需要本地端重運算的場景

## 📝 我的註解與思考

### 文章 hype 的保留意見
標題「AutoCAD 危險!」是典型科技部落格的誇大筆法。實務上:
- Pascal Editor 目前還是**草模推敲工具**等級,離 BIM 產業級標準 (IFC 交換、結構計算、MEP 協調、法規檢核) 還很遠
- AutoCAD/Revit 的真正護城河不是軟體本身,而是**標準化、互通性、法規相容性、業界熟悉度**,這些不是開源就能取代
- WebGPU 雖然瀏覽器支援度提升中,但企業環境的 IT 政策仍是阻礙

### 真正值得學習的點
不是它「能不能取代 AutoCAD」,而是它**示範了一條技術路徑**:
1. 把過去必須 native 才能做的事,用 modern web 重新實作
2. 把 AI 當輔助起稿工具而非全自動生成器
3. 把 GitHub 的協作哲學擴散到非程式領域

這三點對於思考「AI + 開源 + Web」如何重塑專業工具市場,是非常具體的案例。

### 與我的「永續方法學 543」的連結
這個專案某種程度上體現了「**減量 (免授權費) → 透明 (開源) → 可循環 (Fork 文化)**」的數位永續邏輯,可以作為課程中「軟體永續」段落的延伸案例。
