# CC-BOS 論文深度分析

**論文標題：** Obscure but Effective: Classical Chinese Jailbreak Prompt Optimization via Bio-Inspired Search
**發表：** ICLR 2026
**arXiv：** 2602.22983
**機構：** 南洋理工大學、阿里巴巴等 9 家機構

---

## 一、研究核心問題

現有 LLM 的安全對齊（safety alignment）主要以現代語言進行訓練，英文為主，中文也以白話文為主。古典中文（文言文）因語法結構、語義壓縮方式與現代語言截然不同，可能構成安全過濾器的盲區。本研究試圖系統性驗證並利用這個盲區。

## 二、方法論：CC-BOS 框架

CC-BOS 的全名是 Classical Chinese Bio-inspired Optimization Search，核心架構包含三層：

### 2.1 文言文語言層
利用古典中文的語言特性作為基礎混淆手段。文言文的特點包括：一字多義（同一字可為名詞、動詞、形容詞）、省略主語、大量隱喻與典故、高度壓縮的語義結構。這些特性讓安全過濾器難以準確辨識意圖。

### 2.2 八維策略空間
將越獄攻擊提示拆解為八個可組合的維度：

1. **角色身份（Role）**：賦予模型特定角色
2. **行為引導（Behavior）**：引導模型執行特定行為
3. **機制設定（Mechanism）**：設定對話機制規則
4. **隱喻映射（Metaphor）**：用典故或比喻包裝真實意圖
5. **表達風格（Expression）**：控制輸出的語言風格
6. **知識關聯（Knowledge）**：連結特定知識領域
7. **情境設定（Context）**：建構合理的使用場景
8. **觸發模式（Trigger）**：設計啟動有害輸出的觸發條件

每個維度有多個候選選項，形成龐大的組合搜索空間。

### 2.3 果蠅優化演算法（FOA）
使用生物啟發式的果蠅優化演算法在八維策略空間中自動搜索最佳攻擊組合：

- **嗅覺搜索（Smell Search）**：在當前解的鄰域進行局部探索
- **視覺搜索（Visual Search）**：向當前最優方向收斂
- **柯西變異（Cauchy Mutation）**：隨機大幅跳躍，避免陷入局部最優

### 2.4 翻譯模組
由於文言文的回覆難以自動化評估，框架包含一個古典中文轉現代英文的翻譯模組，用於準確判定攻擊是否成功。

## 三、實驗結果

### 3.1 攻擊成功率（ASR）
對六個主流模型進行測試：Gemini、Claude、GPT-4o、DeepSeek、Qwen3、Grok。CC-BOS 達到近乎 100% 的攻擊成功率，平均只需 1.12 至 2.38 次查詢。相比之下，其他攻擊方法需要 3 到 93 次查詢。

### 3.2 消融實驗（Ablation Study）
這是理解這篇論文最關鍵的數據：

| 配置 | ASR |
|------|-----|
| 僅文言文（無策略框架） | 18% |
| 文言文 + 八維策略空間 | 60% |
| 完整 CC-BOS（文言文 + 策略 + 優化） | ~100% |
| CC-BOS 不含翻譯模組 | 90% |
| CC-BOS 含翻譯模組 | ~100% |

### 3.3 跨模型遷移性
用 A 模型找到的攻擊配方，直接用於 B 模型的遷移成功率為 76%–96%。這表示問題不在某一家廠商的安全訓練，而是整個產業的共同盲區。

Meta 的 Llama-Guard-3（專門的安全防護模型）同樣未能有效防禦。

## 四、社群報導 vs 論文實際內容

原始社群貼文的報導大致準確，但有幾處值得釐清：

### 正確的部分
- 六模型全被突破、查詢次數極低、跨模型可遷移——這些描述與論文一致
- 消融實驗的數據引用正確
- 對 Agent 生態風險的延伸討論有見地

### 需要修正或補充的部分
1. **「文言文突破 AI 防守」這個標題有誤導性。** 消融實驗清楚顯示，單獨使用文言文只有 18% 成功率。真正的突破來自三層架構的協同：語言混淆 + 策略組合 + 自動化搜索。文言文是必要條件但遠非充分條件。
2. **翻譯模組的重要性被忽略。** 去掉翻譯模組後 ASR 從 100% 降到 90%，這說明評估精確度對整個框架的最終表現有顯著影響。
3. **「成功率 100%」需要加限定條件。** 這是在特定測試集和評估標準下的結果，並非意味著對任意有害請求都能 100% 成功。

## 五、技術意義與貢獻

### 5.1 揭示系統性盲區
這篇論文最重要的貢獻不是「用文言文騙 AI」，而是系統性地證明了：LLM 安全對齊在低資源語言和古典語言上的覆蓋嚴重不足。這個問題不限於文言文，理論上任何安全訓練覆蓋不足的語言都可能被利用。

### 5.2 攻擊策略的工程化
將越獄攻擊從手工試錯（craft-based）轉變為可自動化搜索的工程問題。八維策略空間的形式化，讓攻擊不再依賴個人靈感，而是可被系統性探索。

### 5.3 極低的查詢成本
平均 1.12–2.38 次查詢就能成功，這意味著：即使有速率限制或異常偵測，攻擊也很難被捕捉。對於實際部署的 AI 系統，這是一個嚴肅的威脅。

## 六、對 Agent 生態的影響

這一點社群貼文分析得相當到位。隨著 Claude Code、OpenClaw（龍蝦）等 Agent 工具的普及，越來越多用戶讓 AI 直接操作工具、處理敏感資料。在 Agent 場景下：

1. **間接提示注入（Indirect Prompt Injection）的風險被放大。** 攻擊者可以在文件、網頁、郵件中嵌入文言文指令，Agent 在處理這些內容時可能被劫持。
2. **工具調用鏈的級聯風險。** 一次成功的注入可能觸發一連串工具操作（刪除檔案、發送郵件、修改程式碼），造成的損害遠超單純的文字生成。
3. **使用者認知落差。** 許多深度使用 Agent 的用戶對提示詞注入的概念一無所知，缺乏基本的防護意識。

## 七、可能的防禦方向

論文本身未提出具體防禦方案，但基於研究結果可推論幾個方向：

1. **語義正規化（Semantic Normalization）**：在安全過濾前，先將輸入轉譯為現代標準語言再進行檢測。但這會增加延遲和成本。
2. **多語言安全訓練擴展**：將安全對齊的訓練數據擴展到更多語言和語體，包括古典語言。但「更多語言」是一場永無止境的軍備競賽。
3. **分層防禦架構**：不依賴單一安全過濾器，而是在多個層級設置檢測——輸入層、推理層、輸出層各自獨立檢查。
4. **意圖層面的安全對齊**：不只比對文字模式，而是在更深的語義/意圖層面進行安全判斷。這在技術上最困難，但也最根本。
5. **Agent 專用防護**：對工具調用設置額外的審查層，特別是涉及不可逆操作（刪除、發送、支付）時，強制人類確認。

## 八、研究局限性

1. **測試範圍有限**：僅測試六個模型，且測試集的規模和覆蓋範圍在社群報導中未充分說明。
2. **ASR 的評估標準**：「成功」的定義影響最終數字。不同的評估標準可能得出不同結論。翻譯模組本身也可能引入評估偏差。
3. **防禦滯後性**：論文發表後，各廠商很可能已經開始針對性修補。這類攻擊的有效性有時效性。
4. **倫理爭議**：公開完整攻擊框架的做法，雖然符合紅隊測試的邏輯，但武器化風險確實存在。

## 九、總評

這是一篇方法論上扎實、問題意識敏銳的研究。入選 ICLR 2026 是合理的。

它的核心價值不在於「文言文能騙 AI」這個吸眼球的表象，而在於：用嚴謹的實驗證明了 LLM 安全對齊的語言覆蓋盲區，並提出了一個高效、可遷移、低成本的自動化攻擊搜索框架。

對於 AI 安全領域，這篇論文是一記警鐘：安全防線不能只盯著主流語言，語言本身就是攻擊面的一部分。
