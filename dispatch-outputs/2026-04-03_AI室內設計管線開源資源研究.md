# AI 室內設計協作管線 — 開源資源與實作案例研究

## 摘要

針對「sketch → AI Vision → JSON → react-planner 2D → SketchUp/FreeCAD → D5/Blender → IFC/SKP/GLB → Notion + Google Drive」這條 AI 室內設計管線，完整調查 30+ 個開源工具和成功案例。核心發現：目前不存在端到端的開源 AI 室內設計管線，但每個環節都有可用的開源工具。最大挑戰是 D5 Render 無法自動化（無 API/CLI），需改用 Blender + Cycles 作為自動化渲染方案。

## 背景

室內設計產業的數位化流程仍高度依賴手動操作和商業軟體。這份研究是為了評估用開源工具建構 AI 協作管線的可行性，找出每個環節最佳的開源選項，並識別關鍵缺口。

## 分析內容

### 管線各階段工具對照

| 管線階段 | 推薦工具 | 替代方案 | 自動化程度 |
|---------|---------|---------|-----------|
| AI Vision → JSON | TF2DeepFloorplan | CubiCasa（商業API） | 高（Flask REST API） |
| 2D 平面編輯 | react-planner | Sweet Home 3D | 中（JSON 資料模型） |
| 3D 建模 | FreeCAD | Blender | 高（Python API + headless） |
| BIM/IFC 處理 | IfcOpenShell + Bonsai BIM | — | 高（完整 Python API） |
| Web 檢視器 | engine_web-ifc / xeokit-sdk | Three.js + IFCLoader | 高 |
| 渲染 | Blender + Cycles | D5 Render（手動） | 高（Blender）/ 無（D5） |
| AI 佈局生成 | HouseDiffusion / Graph2plan | — | 實驗性 |

### 1. AI Vision → 平面圖辨識

**TF2DeepFloorplan**（最佳選擇）
- GitHub: ynqa/TF2DeepFloorplan
- 功能：從平面圖影像辨識房間類型和牆體結構，輸出分割 mask
- 技術：TensorFlow 2，支援 Flask REST API、Docker 部署
- 限制：辨識精度有限，手繪草稿效果不如正式平面圖
- 管線角色：第一步，把設計師草稿轉為結構化資料

**CubiCasa**
- 商業 API 服務，手機拍攝即可生成平面圖
- 精度高於開源方案，但需付費
- 適合需要量產級品質的場景

**FloorplanTransformation**
- GitHub: art-programmer/FloorplanTransformation（ICCV 2017）
- 學術研究性質，將平面圖轉為向量化表示
- 較舊但奠定了這個領域的基礎

### 2. 2D 平面編輯器

**react-planner**
- GitHub: cvdlab/react-planner（~1,400 stars）
- 功能：React 元件庫，基於 JSON 資料模型的 2D 平面圖編輯器
- 優點：JSON 資料模型非常適合程式化操作，可嵌入 Web 應用
- 限制：NPM 套件 6 年未更新，維護狀態堪憂
- 替代方案：work-vv/react-planner（社群復刻版），或 Sweet Home 3D（Java，桌面應用）
- 管線角色：接收 AI 辨識結果（JSON），提供 2D 編輯介面讓設計師調整

### 3. 3D 建模

**FreeCAD**（推薦）
- GitHub: FreeCAD/FreeCAD（29,100+ stars）
- 功能：開源參數化 3D CAD，支援 BIM 工作流（Arch workbench）
- 自動化：完整 Python API，可 headless 運行（伺服器端批次處理）
- IFC 支援：透過 IfcOpenShell 原生匯入匯出 IFC
- 限制：UI 學習曲線陡峭，部分功能不如商業 CAD 穩定
- 管線角色：從 2D 平面圖生成 3D 模型，匯出 IFC/STEP

**Blender**（替代/互補）
- 渲染品質遠勝 FreeCAD
- 透過 Bonsai BIM addon 支援 IFC
- Python scripting 生態更成熟
- 管線角色：3D 渲染 + BIM 整合

### 4. BIM / IFC 處理

**IfcOpenShell**
- GitHub: IfcOpenShell/IfcOpenShell
- 功能：Python/C++ 函式庫，讀寫操作 IFC 檔案
- 核心能力：程式化建立 IFC 模型、解析 IFC 結構、格式轉換
- 管線角色：整條管線的 IFC 資料骨幹

**Bonsai BIM**（原 BlenderBIM）
- GitHub: ~2,400 stars，2026/4 仍活躍更新
- 功能：Blender 內的完整 BIM 工作環境，IFC 原生操作
- 基於 IfcOpenShell，提供 GUI + Python API
- 管線角色：在 Blender 中進行 BIM 建模和 IFC 匯出

### 5. Web 檢視器

**engine_web-ifc**（That Open Company，原 IFC.js）
- IFC.js 已棄用，engine_web-ifc 是正式替代品
- 功能：瀏覽器端 IFC 解析和 3D 渲染
- 管線角色：客戶端 Web 檢視器，讓業主在瀏覽器看 3D 模型

**xeokit-sdk**
- GitHub: xeokit/xeokit-sdk（~890 stars，2026/4 活躍）
- 功能：高效能 Web 3D 檢視器，支援 IFC、glTF、BCF
- 優點：效能最佳，支援大型模型
- 管線角色：高效能替代方案，適合大型專案

### 6. 渲染

**D5 Render**
- **重要發現：D5 沒有 API 或 CLI，無法自動化**
- 只有 LiveSync 外掛（SketchUp、Blender、Rhino 等即時同步）
- 管線角色：只能作為手動精修步驟，不適合自動化管線

**Blender + Cycles**（自動化替代方案）
- 完整 Python scripting 支援，可 headless 批次渲染
- 品質接近商業渲染器
- 管線角色：替代 D5 作為自動化渲染方案

### 7. AI 佈局生成（實驗性）

**HouseDiffusion**（CVPR 2023）
- 從約束條件（房間數量、面積需求）用擴散模型生成平面佈局
- 學術研究，非量產級

**Graph2plan**
- 從約束圖生成平面圖
- 可作為設計師的佈局建議工具

### 成功案例

**E-Interiores**
- 最接近完整管線的案例：22,000 行 Python 程式碼在 Blender 中實現自動化室內設計
- 從平面圖 → 自動佈置家具 → 渲染
- 證明 Blender + Python 路線可行

### 關鍵缺口與風險

1. **端到端整合不存在。** 每個環節有工具，但串接需要大量自訂開發
2. **D5 無法自動化。** 如果堅持用 D5，渲染步驟只能手動。建議改用 Blender + Cycles
3. **react-planner 維護風險。** 核心 repo 停更，需要評估 fork 版本或替代方案
4. **AI 辨識精度。** 手繪草稿 → JSON 的精度仍有限，需要設計師人工修正
5. **IFC 版本相容性。** IFC2x3 vs IFC4 vs IFC4.3 相容性問題需注意

### 建議的技術路線

**開源 BIM 路線（推薦）：**
```
手繪草稿 → TF2DeepFloorplan（AI辨識）→ JSON
    → react-planner（2D 編輯/修正）→ JSON
    → FreeCAD + IfcOpenShell（3D 建模 + IFC）
    → Blender + Bonsai BIM（BIM 整合 + 渲染）
    → IFC/GLB 匯出
    → engine_web-ifc 或 xeokit（Web 檢視器）
    → Notion（專案管理）+ Google Drive（檔案倉庫）
```

**混合路線（商業 + 開源）：**
```
手繪草稿 → CubiCasa API（高精度辨識）→ JSON
    → react-planner（2D 編輯）
    → SketchUp（3D 建模，商業但生態豐富）
    → D5 Render（手動精修渲染）
    → IfcOpenShell（IFC 轉換）
    → SKP/IFC/GLB 匯出
    → Notion + Google Drive
```

### 檔案格式層次

| 層次 | 用途 | 格式 | 來源工具 |
|------|------|------|---------|
| Source（設計師可編輯） | 設計修改 | .FCStd / .blend / .skp | FreeCAD / Blender / SketchUp |
| Delivery（業主可檢視） | 提案展示 | .glb / .pdf / .png | Web 檢視器 / 渲染輸出 |
| Construction（可施工） | 施工圖 | .ifc / .dwg | IfcOpenShell / FreeCAD |

## 結論與建議

開源路線技術上完全可行，但需要大量整合開發工作。建議分階段實施：先打通 react-planner → FreeCAD → Blender 這條核心鏈路，再逐步加入 AI 辨識和自動化渲染。D5 Render 無法自動化是最大的商業限制，如果客戶不堅持 D5 品質，Blender + Cycles 是更好的自動化選擇。

目前不存在端到端的開源 AI 室內設計管線——建構這條管線本身就是開創性的工作。
