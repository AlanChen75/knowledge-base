# MetaClaw 框架分析 — 整合與利用方案

> 針對悠識數位現有工具鏈（Claude Code / Cursor / Antigravity）的整合評估

---

## 一、MetaClaw 是什麼

MetaClaw 是北卡羅來納大學 AIMING Lab 開發的開源 AI Agent 框架。核心概念：在你和 LLM 之間插入一個「透明代理」(transparent proxy)，讓 AI Agent 能夠自我進化——你每次跟它互動，它都在學。

最有趣的設計：它會看你的 Google Calendar，發現你在開會時，就自動開始用閒置的運算資源訓練模型。你開會的時間 = AI 進化的時間。

**GitHub:** [aiming-lab/MetaClaw](https://github.com/aiming-lab/MetaClaw)
**論文:** [arXiv:2603.17187](https://arxiv.org/abs/2603.17187)

## 二、技術架構

### 2.1 透明代理架構

```
你的工具（Claude Code / Cursor / 任何 OpenAI SDK 客戶端）
        │
        ▼
┌─────────────────────────────┐
│   MetaClaw 本地代理 Proxy    │
│   (localhost:port)          │
│                             │
│  ┌───────────┐ ┌─────────┐ │
│  │ 技能注入層 │ │ 對話記錄│ │
│  └───────────┘ └─────────┘ │
│  ┌───────────┐ ┌─────────┐ │
│  │ OMLS 排程器│ │ PRM 評分│ │
│  └───────────┘ └─────────┘ │
└─────────────┬───────────────┘
              ▼
     上游 LLM（Claude / GPT / 開源模型）
```

關鍵：你不需要改任何現有工具的設定，只要把 API endpoint 指向 MetaClaw 的本地 proxy 即可。它同時暴露 OpenAI 相容和 Anthropic 相容（`/v1/messages`）的端點。

### 2.2 三種運作模式

| 模式 | 指令 | 需要 GPU？ | 說明 |
|------|------|-----------|------|
| **Skills-Only** | `metaclaw start --mode skills_only` | 否 | 只做技能注入，不訓練模型權重。最輕量。 |
| **RL 即時訓練** | `metaclaw start --mode rl` | 是（雲端） | batch 滿了就立刻開始 RL 訓練 |
| **Madmax（預設）** | `metaclaw start` | 是（雲端） | 技能 + RL + 智慧排程，閒置/開會時才訓練 |

### 2.3 OMLS 排程器（Opportunistic Meta-Learning Scheduler）

監控三個訊號來判斷「現在可以訓練嗎」：
1. 可設定的休眠時間
2. OS 層級的鍵盤/滑鼠閒置偵測
3. Google Calendar 事件（開會 = 訓練窗口）

### 2.4 雙軌學習機制

**快車道 — 技能驅動適應：** LLM 分析失敗的對話軌跡，自動合成新的行為指令（skill），即時生效，零停機。

**慢車道 — 策略優化：** 用 RL + 過程獎勵模型（PRM）透過雲端 LoRA 微調更新模型權重。

### 2.5 性能數據

把 Kimi-K2.5 從 21.4% 提升到 40.6%，幾乎追平 GPT-5.2 的基準線 41.1%。完全解決的任務數量提升了 8.25 倍。

## 三、跟你現有工具鏈怎麼整合

### 3.1 你目前的工具鏈

```
Claude Code（架構/邏輯）→ Cursor（實作）→ Antigravity + Gemini Flash（驗證）
                ↕ 額度用完就切換 ↕
```

### 3.2 MetaClaw 可以插在哪裡

**場景 A：包住 Claude Code**

```
Claude Code → MetaClaw Proxy → Claude API
```

好處：MetaClaw 會記錄你跟 Claude Code 的所有互動，自動提取「你常犯的錯」和「有效的指令模式」，生成 skill 檔。下次遇到類似任務，它會自動注入這些 skill，相當於 Claude Code 越用越懂你。

限制：Claude 是閉源模型，RL 訓練部分無效。只能用 Skills-Only 模式。但光是技能注入就已經有價值。

**場景 B：包住開源模型做驗證**

```
Antigravity → MetaClaw Proxy → Gemini Flash / 本地開源模型
```

如果你把驗證流程指向的模型換成可微調的開源模型（例如 Qwen2.5 或 Kimi-K2.5），MetaClaw 就能在你開會的時候用 RL 訓練這個模型，讓它越來越擅長你的驗證任務。這是最能發揮 MetaClaw 全部能力的場景。

**場景 C：統一 proxy 管理多工具切換**

你現在「額度用滿就換一套」的痛點，MetaClaw 可以幫忙。它本質上是一個 proxy，可以設定 fallback chain：

```
請求 → MetaClaw Proxy → 先試 Claude → 額度滿了 → 自動切 Gemini → 再滿 → 切開源模型
```

這樣你不用手動切換介面了。

### 3.3 跟 RAG 系統的結合

你正在建的悠識 RAG 知識檢索系統，可以跟 MetaClaw 的 skill 系統形成雙層架構：

- **MetaClaw Skills：** 儲存「怎麼做」的行為知識（prompt 技巧、工作流程、常見錯誤的修正方式）
- **Qdrant RAG：** 儲存「知道什麼」的事實知識（研究報告、專案文件、方法論）

當同事搜尋知識時，MetaClaw 可以用 skill 來優化查詢（等於你一直想打磨的查詢分析層），RAG 提供內容。

## 四、實際整合步驟

### Phase 1：最小可行整合（1-2 天）

```bash
# 安裝
pip install metaclaw

# 初始化設定
metaclaw setup
# → 設定 upstream LLM endpoint（Claude API）
# → 設定 Google Calendar 連接（可選）

# Skills-Only 模式啟動
metaclaw start --mode skills_only

# 把 Claude Code 的 API endpoint 改指向 MetaClaw
# 在 Claude Code 設定中：
# API_BASE_URL=http://localhost:<port>/v1/messages
```

### Phase 2：驗證流程整合（1 週）

把 Antigravity 的驗證流程也接上 MetaClaw，但指向一個可微調的模型。讓 MetaClaw 在閒置時訓練這個模型，使它越來越擅長你的程式碼驗證任務。

### Phase 3：統一 proxy + 自動切換（2 週）

設定 MetaClaw 作為所有 AI 工具的統一入口，配置 fallback chain 和額度監控。

## 五、限制與注意事項

1. **閉源模型無法 RL 訓練。** Claude、GPT 這些只能用 Skills-Only 模式。要用到完整的 RL 訓練能力，得配合開源可微調模型。

2. **Skills-Only 模式的能力上限。** 技能注入本質上是在 system prompt 裡加入學到的指令。如果 system prompt 已經很長，可能會影響效能或撞到 context 上限。

3. **隱私和安全。** MetaClaw 會記錄所有對話。你弟在顧的資安這塊，需要確認：對話 log 存在哪裡？雲端 LoRA 訓練時資料怎麼處理？內網部署是否可以完全離線運作？

4. **框架成熟度。** MetaClaw 還很新，可能有 bug 和 breaking changes。建議先在非關鍵的驗證流程上試水，不要直接用在生產環境。

5. **Google Calendar 整合的實際價值。** 對個人開發者很酷，但對團隊來說，訓練窗口的排程邏輯可能需要自訂。

## 六、跟你現有方式的比較

| 面向 | 你現在的做法 | 加上 MetaClaw 後 |
|------|------------|-----------------|
| 工具切換 | 手動切介面，額度滿了換一套 | Proxy 自動 fallback |
| 經驗累積 | 在你腦裡，每次要重新提示 | 自動提取 skill，持續生效 |
| 驗證優化 | Gemini Flash 通用能力 | 可微調模型越用越準 |
| 開會時間 | 浪費中 | AI 在背景訓練 |
| 複雜度 | 低，靠人肉切換 | 中，多一層 proxy 要維護 |

---

## 七、建議

對你來說，最值得先試的是 **Skills-Only 模式包住 Claude Code**。零成本、零 GPU、最多花一個下午設定。如果你發現 Claude Code 確實「越用越懂你」了，再考慮擴展到 RL 訓練和統一 proxy。

不要一次全上。你現在的多工具交替方式雖然辛苦，但穩定。MetaClaw 應該是「加法」不是「替換」。
