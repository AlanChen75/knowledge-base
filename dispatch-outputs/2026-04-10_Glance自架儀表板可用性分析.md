## 摘要

分析 Glance 這個 Go 語言編寫的輕量自架儀表板專案，評估其作為 Alan 個人基礎設施統一監控面板的可行性。結論：Glance 非常適合作為「資訊聚合 + 輕量監控」的首頁儀表板，部署成本極低，對 Alan 的 sustainability-100 新聞管線、DesignClaw ComfyUI 機器監控、AI 資訊追蹤等需求覆蓋率高。建議以 Docker Compose 部署在 ac-mac 上，搭配 Tailscale 遠端存取。

## 背景

Alan 目前的基礎設施分散在多處：ac-mac 跑 sustainability-100 新聞管線、3090 機器跑 DesignClaw 的 ComfyUI、多個 GitHub repo（Claw 生態系）、SecondBrain 知識庫。缺乏一個統一的「一眼看全局」面板，需要一個低維護成本的 dashboard 來整合資訊流與基本監控。

### 二、核心功能與 Widget 類型

Glance 的設計哲學是「以資訊流為中心」而非「以服務連結為中心」，這使它在 feed 聚合場景有天然優勢。

完整 Widget 清單：

**資訊聚合類**
- RSS：支援任意 RSS/Atom feed，可設 limit、cache、collapse-after
- Hacker News：直接整合，支援排序選項
- Reddit：子版內容，支援縮圖與過濾
- Lobsters：社群連結聚合
- Videos：YouTube 頻道/播放清單追蹤
- Twitch Channels / Top Games：直播追蹤

**監控類**
- Monitor：網站/服務健康檢查（HTTP ping）
- Docker Containers：容器狀態監控
- Server Stats：系統資源監控（CPU、RAM、Disk），支援本地與遠端
- DNS Stats：DNS 分析（搭配 Pi-hole / AdGuard）
- ChangeDetection.io：網頁變更偵測

**工具類**
- Weather：天氣預報（基於地點）
- Markets：股市/加密貨幣行情
- Calendar：日曆事件
- Clock：多時區時鐘
- Search：客製搜尋引擎欄
- Bookmarks：分類書籤連結
- Todo：任務管理
- Releases：GitHub/GitLab release 追蹤
- Repository：Git repo 資訊顯示

**擴展類**
- Custom API：通用 HTTP JSON endpoint，用自訂 HTML 模板渲染
- Extension：從 URL 載入 HTML 內容
- iframe：嵌入外部網頁
- HTML：原生 HTML 內容

**佈局類**
- Group：Widget 容器
- Split Column：並排佈局

### 三、可用性評估

#### 部署難度：⭐ 極低

部署方式三選一：
1. **Docker Compose（推薦）**：一份 `docker-compose.yml` + 一份 `glance.yml` 即可
2. **單一二進位檔**：下載 binary 直接跑，零依賴
3. **第三方**：NixOS、Proxmox VE、Coolify.io 都有支援

不需要資料庫、不需要 Node.js、不需要前端建構工具。這在 dashboard 工具中是少見的極簡架構。

#### 配置複雜度：⭐⭐ 低

- 全部配置在一份 YAML 檔完成
- 支援 `$include` 拆分多個 YAML 模組
- 支援 `${ENV_VAR}` 環境變數注入
- 支援 Docker secrets `${secret:name}`
- 設定檔修改後自動熱重載，不需重啟服務
- 社群有 JSON Schema 提供 IDE 自動補全

#### 擴展性：⭐⭐ 中等

- 優勢：Custom API widget 可對接任何 JSON endpoint，iframe 可嵌入任何網頁
- 優勢：community-widgets 生態持續成長
- 限制：不像 Homepage 有數百個原生 service integration
- 限制：無插件系統，深度客製需要自己寫 HTML 模板或 fork 原始碼

#### 維護成本：⭐ 極低

- 無狀態、無資料庫，壞了重拉就好
- 頁面載入時才抓資料並快取，不做背景輪詢
- AGPL-3.0 授權，自用無限制
- 密碼驗證內建防暴力破解（5 次失敗鎖 5 分鐘）

### 四、對 Alan 需求場景的對應分析

#### 場景 1：監控 sustainability-100 新聞管線（ac-mac）

| 需求 | Glance 方案 |
|------|------------|
| 管線跑在哪 | Server Stats widget 監控 ac-mac 的 CPU/RAM/Disk |
| 管線容器狀態 | Docker Containers widget 顯示容器是否存活 |
| 抓到的新聞 | RSS widget 訂閱管線輸出的 RSS feed（如果有的話）|
| 管線是否卡住 | Monitor widget 定期 ping 管線 health endpoint |

建議：如果管線有 HTTP health endpoint 或產出 RSS feed，Glance 可以直接監控。如果沒有，可以寫一個簡單的 `/health` API 包裝在管線旁邊，或用 Custom API widget 讀取管線 log 最後一行的時間戳。

#### 場景 2：監控 DesignClaw ComfyUI 3090 機器

| 需求 | Glance 方案 |
|------|------------|
| GPU 使用率 | Custom API + nvidia-smi JSON exporter，或 iframe 嵌入 Grafana panel |
| ComfyUI 是否存活 | Monitor widget ping ComfyUI 的 web UI port |
| Docker 容器狀態 | Docker Containers widget（需要 Glance 能連到遠端 Docker socket）|
| 機器 CPU/RAM | Server Stats widget 支援遠端模式 |

建議：Server Stats 支援遠端監控，但 GPU 監控不是內建功能。最務實的做法是在 3090 機器上跑一個小型 JSON API 暴露 `nvidia-smi` 數據，然後用 Custom API widget 渲染。

#### 場景 3：追蹤 AI 相關 RSS / Hacker News

這是 Glance 最強的場景：
- Hacker News widget 直接用，可設定排序和數量
- RSS widget 訂閱 AI 相關 feed（The Batch、Import AI、AI News 等）
- Reddit widget 追蹤 r/MachineLearning、r/LocalLLaMA
- Videos widget 追蹤 AI YouTube 頻道

#### 場景 4：SecondBrain 知識庫狀態

| 需求 | Glance 方案 |
|------|------------|
| 最近更新的筆記 | Custom API：寫一個小 API 讀 SecondBrain git log，回傳 JSON |
| 筆記數量統計 | 同上，`find . -name "*.md" \| wc -l` 包成 API |
| Notion 同步狀態 | Monitor widget ping notion-sync 的 GitHub Actions API |

建議：需要自建一個輕量 API endpoint。可以用一個 10 行的 Go 或 Python 腳本暴露 `/api/secondbrain/stats` JSON，Glance 的 Custom API widget 就能渲染。

#### 場景 5：GitHub Repos 活動（Claw 生態系）

- Releases widget：追蹤多個 repo 的最新 release
- Repository widget：顯示 repo 基本資訊（stars、issues、PR）
- Custom API + GitHub API：拉取跨 repo 的 commit activity

### 五、競品比較

| 面向 | Glance | Homepage | Dashy | Heimdall |
|------|--------|----------|-------|----------|
| 定位 | 資訊流聚合 | 服務入口 + 整合 | 全功能儀表板 | 簡單應用啟動器 |
| 語言 | Go | Next.js | Vue.js | PHP/Laravel |
| Docker Image | < 20 MB | ~200 MB | ~300 MB | ~150 MB |
| 記憶體用量 | < 50 MB | 200+ MB | 200+ MB | 100+ MB |
| 資料庫 | 不需要 | 不需要 | 不需要 | SQLite |
| 配置方式 | YAML | YAML + JSON | YAML + UI | Web UI |
| Service 整合數 | ~20 widgets | 200+ services | 100+ widgets | 有限 |
| Feed 聚合 | ⭐⭐⭐ 核心強項 | ⭐ 有限支援 | ⭐⭐ 部分支援 | ❌ 不支援 |
| 監控能力 | ⭐⭐ 基本 | ⭐⭐⭐ 深度整合 | ⭐⭐ 中等 | ⭐ 極有限 |
| GPU 監控 | 需自建 API | 需自建 API | 需自建 API | 不支援 |
| 行動裝置 | ⭐⭐⭐ 原生響應式 | ⭐⭐ 可用 | ⭐⭐ 可用 | ⭐⭐ 可用 |
| 學習曲線 | 低 | 中 | 中高 | 低 |
| GitHub Stars | 33k | 22k+ | 18k+ | 7k+ |

**選擇建議**：
- 如果主要需求是「一早打開瀏覽器看到所有資訊流」→ **Glance 最佳**
- 如果需要深度整合幾十個 homelab 服務（Plex、Sonarr、Pi-hole 等）→ **Homepage 更適合**
- 如果想要視覺豐富 + 大量自訂 widget → **Dashy**
- Heimdall 功能已經落後，不建議新部署

對 Alan 的場景，Glance 覆蓋了 70% 需求（RSS、HN、監控、GitHub），剩下 30%（GPU 監控、SecondBrain 狀態）需要自建小型 API，但 Custom API widget 讓這件事很簡單。

## 結論與建議

### 推薦方案：Glance + 輕量輔助 API

Glance 是 Alan 需求場景的最佳選擇，原因：
1. 極低部署和維護成本，符合一人團隊的時間預算
2. Feed 聚合是核心強項，完美對應 AI 資訊追蹤需求
3. 單一 YAML 配置，可以 git 管理，符合 SecondBrain 管理哲學
4. Docker image < 20MB、記憶體 < 50MB，ac-mac 上可以無感運行

### 建議部署架構

```
ac-mac (Docker)
├── glance (port 8080)
│   └── glance.yml (配置)
├── sb-stats-api (port 8081) ← 自建：SecondBrain stats JSON API
└── Tailscale ← 遠端安全存取

3090 機器 (Docker)
├── comfyui
├── gpu-stats-api (port 8082) ← 自建：nvidia-smi JSON exporter
└── Tailscale
```

### 輔助 API 範例（SecondBrain Stats）

```python
# sb-stats-api.py — 10 行的 SecondBrain stats endpoint
from flask import Flask, jsonify
import subprocess, os
app = Flask(__name__)
SB_PATH = os.getenv("SB_PATH", "/path/to/SecondBrain")

@app.route("/health")
def health():
    return "ok"

@app.route("/stats")
def stats():
    total = subprocess.check_output(
        f"find {SB_PATH} -name '*.md' | wc -l", shell=True
    ).decode().strip()
    last = subprocess.check_output(
        f"git -C {SB_PATH} log -1 --format='%ai'", shell=True
    ).decode().strip()
    month_new = subprocess.check_output(
        f"find {SB_PATH} -name '*.md' -newer {SB_PATH}/.git -mtime -30 | wc -l",
        shell=True
    ).decode().strip()
    return jsonify(total_notes=int(total), last_updated=last, month_new=int(month_new))

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8081)
```

### 輔助 API 範例（GPU Stats）

```python
# gpu-stats-api.py — nvidia-smi JSON exporter
from flask import Flask, jsonify
import subprocess, xml.etree.ElementTree as ET
app = Flask(__name__)

@app.route("/gpu")
def gpu():
    xml = subprocess.check_output(["nvidia-smi", "-q", "-x"]).decode()
    root = ET.fromstring(xml)
    gpu = root.find("gpu")
    return jsonify(
        gpu_name=gpu.find("product_name").text,
        gpu_util=gpu.find("utilization/gpu_util").text.replace(" %", ""),
        mem_used=gpu.find("fb_memory_usage/used").text,
        mem_total=gpu.find("fb_memory_usage/total").text,
        temperature=gpu.find("temperature/gpu_temp").text.replace(" C", ""),
    )

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8082)
```
