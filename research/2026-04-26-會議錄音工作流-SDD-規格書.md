# 個人會議錄音 AI 摘要基礎設施（SDD 規格書）

（以下為原始內容，保留供歷史參考）

> **可作為課程教材引用**
> 適用課程：AI 四級課程（進階模組）、AI 智慧製造三級課程、AI 永續兩級課程
> 教學定位：學員 6-8 小時課程內完成自己的 MVP 版本，理解「模組化 MVP 基礎設施」哲學

## 📌 摘要

打造手機瀏覽器 PWA 錄音 → 即時逐字稿 → AI 摘要 → 自動分發（GitHub / Email / Notion）的個人會議基礎設施。設計同時兼顧個人實用與課程教學雙重目標：每個元件都可替換、可拆解、可作為教學單元。

## 🎯 設計目標

1. **個人實用**：錄完會議 5 分鐘內，結構化筆記已進知識庫
2. **資料主權**：客戶機密內容不留存第三方廠商（除 STT API 短暫處理）
3. **成本可控**：每月使用成本 < 30 USD
4. **教學友善**：每個元件可替換、有清楚階段切分（MVP → 進階 → 客製化）
5. **生態整合**：複用既有 SecondBrain MCP + Cloudflare 帳號 + 課程基礎設施

---

## 1️⃣ Proposal（提案）

### 為什麼要做這個

**個人需求**：
- 經常開會、講課、做企業培訓，需要快速產出會議紀錄
- 既有方案（MeetGeek、Otter）需付費且資料跑在第三方，不適合政府/企業客戶機密場景
- 既有 SecondBrain MCP + GitHub 知識庫架構已就位，缺一個「上游入口」

**教學需求**：
- 「AI 四級課程」進階模組需要實戰專案教材
- 學員學完應該能「打造自己的工具」，不是「教學員怎麼買工具」
- 此專案橫跨 PWA、API 整合、雲端部署、AI Agent，涵蓋 AI 四級課程多個層級

**戰略需求**：
- 可成為第五個 MVP 網站基礎設施（呼應既有 Ai100講、永續100講、法拍地圖、保險資訊全球站）
- 未來可包裝為「永續會議筆記平台」對外推出（選擇性）

### 不做什麼（明確排除）

- ❌ 視訊會議 bot（Zoom/Teams 機器人加入會議）——複雜度過高且非核心需求
- ❌ 完整使用者系統（註冊、權限、計費）——MVP 階段個人使用為主
- ❌ Speaker diarization（區分不同講者）——進階功能，列為 v2
- ❌ 跨會議搜尋與分析——v2 功能
- ❌ iOS/Android 原生 App——PWA 已足夠

### 預期成果

| 階段 | 時程 | 產出 |
|------|------|------|
| MVP | 1-2 週 | 可在手機瀏覽器錄音、產逐字稿、自動推到 GitHub |
| 進階 | 第 3-4 週 | 加入 Email 分發、Notion 整合、即時逐字稿顯示 |
| 課程化 | 第 5-6 週 | 拆解成 6-8 小時課程模組、學員手冊、範例 repo |
