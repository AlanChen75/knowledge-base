# 個人會議錄音 AI 摘要基礎設施（SDD 規格書）

> **可作為課程教材引用**
> 適用課程：AI 四級課程（進階模組）、AI 智慧製造三級課程、AI 永續兩級課程
> 教學定位：學員 6-8 小時課程內完成自己的 MVP 版本，理解「模組化 MVP 基礎設施」哲學

## 📌 摘要

打造手機瀏覽器 PWA 錄音 → 即時逐字稿 → AI 摘要 → 自動分發（GitHub / Email / Notion）的個人會議基礎設施。設計同時兼顧個人實用與課程教學雙重目標：每個元件都可替換、可拆解、可作為教學單元。

## 🎯 設計目標

1. **個人實用**：錄完會議 5 分鐘內，結構化筆記已進知識庫
2. **資料主權**：客戶機密內容不留存第三方廠商（除 STT API 短暫處理）
3. **成本可控**：每月使用成本 < 30 USD
4. **教學友善**：每個元件可替換、有清楚階段切分（MVP → 進階 → 客製化）
5. **生態整合**：複用既有 SecondBrain MCP + Cloudflare 帳號 + 課程基礎設施

## 1️⃣ Proposal（提案）

### 為什麼要做這個

**個人需求**：
- 經常開會、講課、做企業培訓，需要快速產出會議紀錄
- 既有方案（MeetGeek、Otter）需付費且資料跑在第三方，不適合政府/企業客戶機密場景
- 既有 SecondBrain MCP + GitHub 知識庫架構已就位，缺一個「上游入口」

**教學需求**：
- 「AI 四級課程」進階模組需要實戰專案教材
- 學員學完應該能「打造自己的工具」，不是「教學員怎麼買工具」
- 此專案橫跨 PWA、API 整合、雲端部署、AI Agent，涵蓋 AI 四級課程多個層級

**戰略需求**：
- 可成為第五個 MVP 網站基礎設施（呼應既有 Ai100講、永續100講、法拍地圖、保險資訊全球站）
- 未來可包裝為「永續會議筆記平台」對外推出（選擇性）

### 不做什麼（明確排除）

- ❌ 視訊會議 bot（Zoom/Teams 機器人加入會議）——複雜度過高且非核心需求
- ❌ 完整使用者系統（註冊、權限、計費）——MVP 階段個人使用為主
- ❌ Speaker diarization（區分不同講者）——進階功能，列為 v2
- ❌ 跨會議搜尋與分析——v2 功能
- ❌ iOS/Android 原生 App——PWA 已足夠

### 預期成果

| 階段 | 時程 | 產出 |
|------|------|------|
| MVP | 1-2 週 | 可在手機瀏覽器錄音、產逐字稿、自動推到 GitHub |
| 進階 | 第 3-4 週 | 加入 Email 分發、Notion 整合、即時逐字稿顯示 |
| 課程化 | 第 5-6 週 | 拆解成 6-8 小時課程模組、學員手冊、範例 repo |

## 2️⃣ Design（系統設計）

### 整體架構

```
┌─────────────────────────────────────────────────────────────┐
│  手機端 PWA（瀏覽器）                                          │
│  - MediaRecorder API 錄音                                     │
│  - WebSocket 串流 / 分段 HTTP 上傳                            │
│  - Service Worker（離線可錄、自動補傳）                        │
└─────────────────────┬───────────────────────────────────────┘
                      │ HTTPS / WSS
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  Cloudflare Workers（API Gateway + 業務邏輯）                 │
│  - Token-based 認證                                           │
│  - 音訊分段轉發給 STT                                          │
│  - 收集完整逐字稿後呼叫 Claude API                             │
│  - 觸發分發 webhook                                            │
└─────┬─────────────────┬─────────────────┬──────────────────┘
      │                 │                 │
      ▼                 ▼                 ▼
┌───────────┐    ┌─────────────┐    ┌──────────────────────┐
│ Deepgram  │    │ Claude API  │    │ 分發層                 │
│ (主) 即時  │    │ Sonnet 4.5  │    │ - GitHub (SecondBrain) │
│ STT       │    │ 產摘要      │    │ - Email (Resend)       │
│           │    │ + Action    │    │ - Notion API           │
│ Whisper   │    │ Items       │    │ - Slack（選擇性）      │
│ (備) 本機  │    └─────────────┘    └──────────────────────┘
└───────────┘
```

### 元件規格

#### A. 手機端 PWA

**技術選型**：
- **HTML + Vanilla JS + Tailwind**（教學友善，不需框架；學員不會被 React 學習曲線卡住）
- **可選升級**：學員若想學 React 可改用 Vite + React 作為進階教材

**核心 API**：
```javascript
// 錄音
navigator.mediaDevices.getUserMedia({ audio: true })
new MediaRecorder(stream, { mimeType: 'audio/webm;codecs=opus' })

// 即時上傳（兩種模式擇一）
// 模式 A: WebSocket 串流（即時逐字稿）
const ws = new WebSocket('wss://your-worker.workers.dev/stream')

// 模式 B: 分段 HTTP（每 5 秒切一段，較簡單）
recorder.ondataavailable = (e) => uploadChunk(e.data)
```

**PWA 必備**：
- `manifest.json`（App icon、名稱、開啟模式）
- Service Worker（離線可錄、網路恢復自動補傳）
- `Add to Home Screen`（手機桌面圖示，看起來像原生 App）

**UI 最小可行版**：
- 一個大圓形按鈕：開始 / 停止錄音
- 即時顯示錄音時長
- 即時顯示逐字稿（捲動）
- 結束後顯示「處理中」→「已推送到 X」

#### B. Cloudflare Workers 後端

**為什麼選 Workers**：
- 你已熟悉（保險資訊全球站用過類似架構）
- 免費額度：每天 10 萬次請求，個人使用足夠
- 全球邊緣節點，台灣延遲低
- 可直接綁定 D1（SQLite）、KV、R2，無需另外部署資料庫

**主要 endpoints**：
```
POST /api/sessions/start       建立新會議 session
POST /api/sessions/:id/audio   上傳音訊片段
GET  /api/sessions/:id/transcript  取得逐字稿（即時）
POST /api/sessions/:id/end     結束會議，觸發摘要與分發
GET  /api/sessions/:id/result  取得最終結果
```

**儲存策略**：
- D1：session metadata（id、開始時間、狀態、推送目的地）
- R2：原始音訊（選擇性，預設不存以保資料主權）
- KV：暫存逐字稿片段（會議結束後刪除）

#### C. STT 引擎（雙軌制）

**主軌：Deepgram**
- 為什麼：即時性最強、中文準確度好、價格便宜（$0.0043/分鐘）
- 中文模型：`nova-2-general` 或 `nova-3-general`（持續更新）
- WebSocket 連線：邊錄邊轉，延遲 < 500ms
- 1 小時會議約 $0.26 USD

**備軌：Whisper.cpp（MacBook 本機）**
- 為什麼：完全離線、零成本、極致資料主權
- 缺點：非即時，需要會議結束後上傳
- 適合：政府/企業機密會議、敏感客戶對話
- 教學價值：示範「完全本機」的 AI 工作流選項

**選擇邏輯**：
- 一般會議 → Deepgram（即時）
- 機密會議 → 切換至 Whisper 模式（會議結束後本機處理）
- UI 上提供切換開關

#### D. Claude API 摘要產生

**模型**：Claude Sonnet 4.5（性價比最佳）
- 個人重要會議可升級到 Opus 4.5

**Prompt 結構**（範本，可由使用者自訂）：
```
你是專業會議紀錄助理。請根據以下逐字稿產出：

1. 會議基本資訊（時間、估計參與人數、會議類型）
2. 結構化摘要（依議題分段）
3. 關鍵決議（明確列出做出的決定）
4. Action Items（負責人、待辦、截止日期；無法判斷時標 TBD）
5. 待釐清問題
6. 下次會議預告

輸出格式：Markdown，使用繁體中文，符合台灣商務語境。
```

**成本**：1 小時會議約 $0.05-0.15 USD

#### E. 分發層

**GitHub（複用 SecondBrain MCP）**：
- 推到 `research/meeting-notes/YYYY-MM-DD-會議名稱.md`
- 自動加 frontmatter（含 tags、category、source）

**Email（Resend）**：
- 為什麼選 Resend：API 簡單、有免費額度、開發者友善
- 模板：摘要在 body、完整逐字稿在 attachment

**Notion API（選擇性）**：
- 適合學員若主要使用 Notion 而非 GitHub
- 教學價值：示範同一份內容兩種儲存目的地

### 安全性考量

1. **API token**：手機 PWA 用 token 驗證，token 存在 localStorage（個人使用接受）
2. **CORS 限制**：Workers 只接受白名單 origin
3. **音訊不持久化**：預設不存 R2，會議結束後刪除暫存
4. **HTTPS 強制**：Workers 自動提供，PWA 需 HTTPS 才能用麥克風
5. **STT 服務選擇**：Deepgram 有 SOC 2、不留訓練資料聲明

## 4️⃣ 教學設計（給課程使用）

### 學習目標

學員學完應能：
1. 打造自己的手機 PWA 應用（理解 MediaRecorder、Service Worker）
2. 部署 serverless 後端（Cloudflare Workers）
3. 整合多個 AI API（STT + LLM）
4. 設計 webhook 與分發機制
5. **更深層**：理解「模組化 MVP 基礎設施」的哲學——每個元件都是可替換的

### 教學重點（呼應你既有的論述線）

- **「不要做免費仔」的延伸**：自建 ≠ 省錢，是換取資料主權與客製能力
- **模組化哲學**：示範 Deepgram → Whisper、Cloudflare → Vercel、Claude → Gemini 的替換
- **AI Agent 進階**：這個系統本質是「會議專屬 agent」，可串接其他 agent（如自動排定下次會議、自動建立 Jira 任務）
- **harness engineering**：作為你 agent 編排架構思考的具體案例

### 課程定位建議

| 課程 | 切入角度 |
|------|---------|
| **AI 四級課程** | 第三/四級進階模組——「打造你的 AI 工具基礎設施」 |
| **AI 智慧製造三級課程** | 應用案例：工廠會議、產線檢討會、客戶需求訪談 |
| **AI 永續兩級課程** | 應用案例：ESG 委員會會議、永續報告書訪談、利害關係人會議 |
| **企業培訓** | 1 天工作坊：管理階層理解 + 工程團隊實作 |

### 對自己工作流的價值

1. **學術論文工作流**：採訪業界專家、與指導教授討論時可即時錄音轉錄
2. **企業培訓**：講課時開錄音，課後直接產出講義初稿
3. **客戶會議**：政府/企業客戶會議直接走 Whisper 本機模式，零洩漏風險
4. **知識管理**：每場會議自動進知識庫，未來可被其他 AI 檢索

### 對教學的價值

1. **完整 vertical slice**：從手機 UI 到雲端後端到 AI 整合，學員一路看下來
2. **實用性**：學員學完馬上有自己的工具，不是 toy project
3. **可複製、可客製**：每個學員的版本都會不同（不同 STT、不同分發目的地、不同 prompt）
4. **與既有課程體系完美互補**：補足「實戰專案」這塊空缺

### 對網站基礎設施的擴展

- 可成為第五個 MVP 網站：「會議筆記基礎設施」
- 與既有四個網站共用 Cloudflare 帳號、共用部署 pipeline
- 未來可對外提供（白標版本給企業客戶）

## 📝 我的註解與思考

**這個專案的真正價值不在「會議錄音」**，而在「**這是學員第一個能完整擁有的 AI 基礎設施**」。市面上的會議工具都把使用者鎖在他們的雲端；自己打造的版本，學員學會的不是「怎麼用工具」，而是「怎麼造工具」。這呼應我長期主張的「不要做免費仔」——但層次更高：不只是付費用最強的，而是**自己變成造工具的人**。

**與 harness engineering 的連結**：這個系統其實是一個「會議專屬 agent」，包含 perception（錄音 + STT）、memory（知識庫）、reasoning（Claude API 摘要）、action（分發）四個經典 agent 元件。教學上可以作為 agent 架構的具體入門案例。

**戰略上的副產品**：當這個系統穩定後，我自己每次會議都會有結構化筆記進入 SecondBrain 知識庫——這個語料庫長期累積後，可以餵回 RAG 系統，變成「我的知識體外延伸」。這是比工具本身更深的價值。

**為什麼不買現成**：除了資料主權與成本，最重要的是「**買了現成就失去教學素材**」。我做課程的核心競爭力是「我自己每天在做的事 → 變成教學內容」。買 MeetGeek 我就只能教「怎麼用 MeetGeek」；自建我可以教「怎麼造 MeetGeek」。這是兩個完全不同層次的課程價值。
