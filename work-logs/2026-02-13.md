### 目標

### 硬體資訊
- **CPU**: Intel i5-2520M @ 2.50GHz (4 threads)

- **GPU**: Intel HD 3000 + AMD Radeon HD 6630M (TURKS)
- **WiFi**: Broadcom 4331
- **OS**: Ubuntu 22.04.5 LTS (kernel 6.8.0-40-generic HWE)

### 問題與解決

#### 1. 顯示驅動 — 系統無法正常開機
- **症狀**: 安裝 Ubuntu 後無法正常開機看到畫面，只能用 Live USB safe mode 進入
- **根因**: radeon 驅動初始化後系統凍結（GPU VBIOS 讀取失敗 Invalid PCI ROM header signature + DPM 鎖死）
- **嘗試**:
  - radeon.dpm=0 radeon.audio=0 → 方塊亂碼
  - radeon.modeset=0 i915.modeset=1 → 未測試（改為保守方案）
  - nomodeset + Wayland → 有綠線但可用
  - nomodeset + X11 → 有綠線但可用
- **最終方案**: GRUB 加 nomodeset，GDM 強制 X11 (WaylandEnable=false)
- **綠線結論**: AnyDesk 遠端畫面正常，確認為 HDMI 硬體接口問題，非軟體問題
- **變更檔案**:
  - /etc/default/grub — 加 nomodeset，GRUB_TIMEOUT=5，GRUB_TIMEOUT_STYLE=menu
  - /etc/gdm3/custom.conf — WaylandEnable=false, AutomaticLogin=ac-2012

#### 2. WiFi 驅動
- **症狀**: Broadcom 4331 缺少 b43 韌體 (ucode29_mimo.fw not found)
- **修復**: 透過 Live USB chroot 安裝 firmware-b43-installer
- **結果**: WiFi 正常連線 Ruby 網路

## [23:30] tg-monitor-bot 錯誤修復 (ac-mac)
- **來源**: MacBook CLI
- **狀態**: ✅ 完成

### 問題
tg-monitor-bot.service 在處理 Happy Coder 任務時頻繁出現錯誤 log。

### 根因分析

#### 1. LimitOverrunError — asyncio StreamReader 緩衝區溢出
- **症狀**: `ValueError: Separator is found, but chunk is longer than limit`
- **根因**: Claude CLI 的 `--output-format stream-json` 輸出單行超過 asyncio.StreamReader 預設的 64KB 限制
- **修復**: 在 `/usr/local/bin/server-monitor/happy_coder.py` 第 232 行的 `create_subprocess_exec` 加入 `limit=1024*1024`（1MB）
- **變更檔案**: `/usr/local/bin/server-monitor/happy_coder.py`

#### 2. systemd StartLimitIntervalSec 位置錯誤
- **症狀**: `StartLimitIntervalSec= in [Service] section, moving to [Unit]`
- **根因**: `StartLimitIntervalSec` 和 `StartLimitBurst` 放在 `[Service]` section，應該在 `[Unit]`
- **修復**: 重寫 `/etc/systemd/system/tg-monitor-bot.service`，將參數移至 `[Unit]` section
- **變更檔案**: `/etc/systemd/system/tg-monitor-bot.service`

### 結果
服務重啟後正常運行，錯誤消除。

### [22:40] 修復 Gemini Image API 舊圖片 bug

**來源**: MacBook CLI
**狀態**: ✅ 完成

**問題**: TG Bot /image 指令生成圖片後，回傳的是上一次的舊圖而非新生成的圖
**根因**:
1. 只在 URL 不含 /app 時才導航到新對話，舊對話不會重置
2. 圖片偵測不分新舊，找到任何 googleusercontent 圖就回傳
3. 點擊圖片截圖時 DOM 元素脫離（Gemini 重渲染）

**修復** (gemini_image_api.py):
1. 每次生成前一律 goto 新對話頁面
2. 發 prompt 前記錄頁面已有圖片 src 集合，只接受新 src
3. 改用 urllib 直接下載圖片 URL，避免 DOM 操作

**測試**: 連續生成 2 張不同主題圖片，均正確生成並推送到 Telegram
