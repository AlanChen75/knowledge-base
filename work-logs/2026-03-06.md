
## [13:00] AI Hub v2.4.0: Firefox → Chrome CDP 大遷移

- **來源**: MacBook CLI
- **Session**: 續接 3/5 session
- **狀態**: ✅ 完成

### 問題根因
- Firefox cookie 複製方案被 Google 指紋檢測擊敗
- Snap Firefox 148 的 cookies 被 Playwright Firefox 146 使用時，Google server-side 拒絕
- UA Override 測試也失敗 — Google 不只看 UA，還有 TLS/JS fingerprint 差異
- Cookie 值完全一致（md5 比對確認），但 Google 就是不認

### 解決方案：Chrome + CDP + Auto-login
- Playwright 連接到真實 Chrome (145.0.7632.116) 的 CDP port
- 同一個 Chrome 登入 + 操作，沒有指紋不匹配
- .env 帳密自動登入，Session 保存在 user-data-dir
- Chrome 版本被 apt-mark hold 保護，不會自動更新

### 變更檔案
-  — 全新 ChromeManager v2 (CDP + auto-login + get_page API)
-  — 切換到 Chrome backend
-  — 修復 audio 提取 (video tag with usercontent URL)
-  — 保留但不再使用

### 測試結果
- gemini_image: ✅ 64s (含 Chrome 啟動 + 自動登入)
- gemini_audio: ✅ 38s (Pro 帳號正常，Firefox 上無法運行的問題解決)
- auto-login: ✅ 偵測到需要登入 → 自動填入帳密 → 登入成功

## [20:30] AEO Monitor — AI 搜尋引擎曝光度監控系統

- **Gemini**: 透過 AI Hub API (Groq LLM fallback)
- **CCBot**: Common Crawl CDX API
