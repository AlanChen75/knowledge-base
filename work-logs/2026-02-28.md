## [20:00] AI Hub 圖片生成故障排查與修復

- **來源**: MacBook CLI
- **狀態**: ✅ 完成

### 問題描述
AI Hub 的 Gemini 圖片生成服務無法運作。Playwright `connect_over_cdp` 連線 Chrome CDP 時 hang 住 timeout。

### 根因
- ac-mac 重啟後 kernel 從 **6.8.0-94** 自動升級到 **6.8.0-101**
- 新 kernel 導致 Chrome 145 的 CDP 協議不相容
- 已知 Playwright issues: #35928, #35115

### 解決方案：Kernel 回滾
```bash
sudo grub-reboot 'gnulinux-advanced-..>gnulinux-6.8.0-94-generic-advanced-...'
sudo reboot
```
回滾到 6.8.0-94 後，CDP 連線立即恢復正常。

### 已嘗試的替代方案（記錄備用）

#### Playwright + Firefox + Cookie 注入（可行）
1. 從 snap Firefox 的 `cookies.sqlite` 讀取 Google cookies
   - Profile: `~/snap/firefox/common/.mozilla/firefox/8oxk9x2z.default`
   - 關鍵：expiry 是毫秒，注入 Playwright 時需除以 1000
2. 注入到 Playwright Firefox context (`ctx.add_cookies()`)
3. Gemini 登入成功、圖片生成成功（40s）
4. 測試腳本: `/tmp/test_ff_fix.py`, `/tmp/test_ff_img2.py`
