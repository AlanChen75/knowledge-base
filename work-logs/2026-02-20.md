## [22:10] Podcast Tracker Download Fix — CDP Download Behavior

- **來源**: MacBook CLI
- **狀態**: ✅ 完成

### 問題
Podcast tracker 在 NotebookLM 頁面點擊「更多」→「下載」後，Chrome 沒有實際下載檔案到 ~/下載/ 目錄。

### 根因分析
1. **CDP 下載行為未設定**: Playwright 透過 CDP 連接的 Chrome，預設不允許下載。必須透過 CDP 設定 `Browser.setDownloadBehavior` 為 `allow` 才能啟動下載。
2. **More 按鈕選錯目標**: `_find_audio_more_btn` 找到的第一個 `aria-label="更多"` 按鈕是影片項目的，不是音訊項目的。
3. **檔案覆寫偵測遺漏**: `_wait_for_chrome_download` 只偵測新檔案，不偵測已存在檔案被覆寫。當 Chrome 覆寫同名檔案時，偵測失敗。
4. **git push 超時**: 25MB 音訊檔案推送需要超過 30 秒。
5. **狀態保存順序**: state 在 deploy 之後才寫入，deploy 崩潰導致狀態丟失。

### 修正
- `_enable_cdp_downloads()`: 新增 CDP session，設定 `Browser.setDownloadBehavior` 為 `allow`
- `_find_audio_more_btn()`: 使用 JS 尋找 `audio_magic_eraser` 項目旁邊的 More 按鈕
- `_wait_for_chrome_download()`: 同時偵測新檔案和覆寫檔案（比對 mtime）
- git push timeout: 30s → 120s
- 狀態寫入順序: 先寫 state，再 deploy（deploy 用 try/except 包裝）
- `requeue_stuck_episodes()`: 新增 `topic["status"] = "pending"` 讓 pipeline 重新處理
