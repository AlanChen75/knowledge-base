# PaddleOCR-VL-1.5 發布：0.9B 輕量級文件視覺語言模型

## 摘要

百度發布 PaddleOCR-VL-1.5，一款專門解析文件的視覺語言模型。模型大小僅 0.9B，卻在 OmniDocBench v1.5 上達到 94.5% 準確率，推理速度比 MinerU 2.5 快 14%、比 dots.ocr 快兩倍以上。即便與 235B 的 Qwen3-VL 相比，在文件解析任務上依然不遜色。

## 關鍵要點

### 模型架構
- **大小**: 0.9B 參數（僅 900MB 等級）
- **視覺編碼器**: NaViT 動態高解析度視覺編碼器
- **語言模型**: ERNIE-4.5-0.3B
- **兩階段流程**:
  1. PP-DocLayoutV3 版面分析與不規則形狀定位
  2. 精細元素識別

### 核心能力
- 文字辨識（OCR）
- 表格解析
- 數學公式辨識
- 圖表分析
- 印章識別（新增功能）
- 文字定位（新增功能）
- **輸出格式**: 結構化 Markdown 與 JSON

### 準確率與效能
| 基準測試 | 準確率 |
|---------|-------|
| OmniDocBench v1.5 | 94.5% |
| Real5-OmniDocBench | 92.05% |

| 推理速度對比 | 提升幅度 |
|-------------|---------|
| vs MinerU 2.5 | 快 14% |
| vs dots.ocr | 快 2 倍以上 |

- **語言支援**: 111 種語言
- **與 Qwen3-VL (235B) 相比**: 文件解析任務不遜色

### 真實場景支援（Real5-OmniDocBench）
首創針對真實環境的基準測試，涵蓋：
1. **掃描瑕疵**: 髒污、破損、模糊
2. **傾斜文件**: 拍攝角度偏斜
3. **彎曲文件**: 書籍、摺頁
4. **螢幕翻拍**: 手機拍螢幕
5. **光照不均**: 陰影、反光

在這五種場景下仍達 **92.05%** 準確率。

## 技術優勢

### 1. 輕量高效
- 0.9B 參數可在消費級 GPU 運行
- 推理速度顯著優於同類模型
- 適合本地部署與邊緣運算

### 2. 真實場景適應性
- Real5-OmniDocBench 專為實際使用設計
- 處理低品質、傾斜、彎曲文件能力強
- 手機翻拍螢幕仍能保持高準確率

### 3. 多功能整合
- 一個模型同時處理文字、表格、公式、圖表
- 新增印章識別（對中文文件重要）
- 支援文字定位（可用於版面分析）

### 4. 結構化輸出
- 直接輸出 Markdown 與 JSON
- 便於後續處理與整合

## 應用場景

- **文件數位化**: 掃描書籍、論文、報告
- **表單處理**: 自動提取結構化資料
- **學術論文解析**: 公式、圖表、表格辨識
- **行動端 OCR**: 手機拍照即時辨識
- **合約分析**: 印章、簽名、文字提取
