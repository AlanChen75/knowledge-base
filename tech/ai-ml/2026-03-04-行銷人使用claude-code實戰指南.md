# 行銷人使用 Claude Code 實戰指南

## 📊 元資訊

- **來源類型**：文章/心得分享

## 📌 摘要
這是一篇針對非工程師背景的行銷人員，如何有效使用 Claude Code 的實戰指南。強調不需要精通程式碼，但需要理解基礎概念，並保持「人在迴路中」(Human-in-the-loop) 的工作模式。

## 🏷️ 標籤分類
- **大分類**：tech
- **小分類**：ai-ml

## 🔑 關鍵要點
1. **懂基礎不代表要會寫程式**：了解 HTML/CSS/JS 的角色、框架基礎、SEO 原理、部署概念
2. **Human-in-the-loop 原則**：AI 做苦差事（草稿、數據整理），人負責品牌語氣、策略決策、編輯判斷、創意方向
3. **請工程師把關**：涉及產品或面向用戶的改動，發布前必須請工程師 review
4. **自動化重複性工作**：定期報表、整理試算表、檢查排名等崩潰日常可以交給 AI
5. **誠實面對缺點**：有學習曲線、不一定更快、會犯錯、不能取代策略思考

## 💬 金句摘錄
> "那些說『完全不用懂程式碼』的人是在騙你。"
> — 原文作者

> "AI 傾向找『最簡單的路徑』解決問題，但最簡單不等於最好，有時候反而會惹麻煩。"

> "行銷需要個性，AI 沒有個性。"

## 🧠 概念連結
- **ELI5（Explain Like I'm 5）**：請 AI 用五歲小孩能懂的方式解釋，降低學習門檻
- **Staging 環境**：測試空間，跟正式網站一樣但用戶看不到，用來驗證改動
- **Human-in-the-loop**：人機協作模式，AI 輔助但人類保持控制權

## 💡 與我的連結
- 這正是我使用 Claude Code 和 Happy Coder 的方式：讓 AI 處理繁瑣的程式碼工作，我專注在決策和方向
- 「請工程師把關」這點很重要，雖然我自己有一定技術背景，但複雜改動還是要謹慎
- 工具清單可以參考，看看 Firecrawl、Composio 是否對我的工作流有幫助

## 📝 我的註解與思考
這篇文章的價值在於「誠實」——不是一味鼓吹 AI 萬能，而是清楚說明限制和風險。特別認同「重點是獨立性，不需要等別人排程」這個觀點。對於小團隊或個人來說，能自己處理活動頁面和簡單自動化，確實能大幅提升效率。

但要注意的是：這種工作方式更適合有一定技術理解力的人。完全零基礎的行銷人可能需要更多時間適應。
