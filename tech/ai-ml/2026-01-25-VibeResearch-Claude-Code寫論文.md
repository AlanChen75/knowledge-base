# VibeResearch - Claude Code 自動撰寫學術論文

## 📊 元資訊

## 📌 摘要
史丹佛大學教授 Andy Hall 使用 Claude Code 在不到一小時內成功擴展了一篇已發表的政治科學論文。經過人工驗證，Claude 的準確率極高（29/30 縣正確編碼，數據相關係數 > 0.999），僅有三個小錯誤，類似人類首次撰寫時可能犯的錯誤。

## 🏷️ 標籤分類
- **大分類**：tech
- **小分類**：ai-ml

## 🔑 關鍵要點
1. **Claude Code 完成的任務**：下載程式庫、轉換 Stata→Python、網路爬取數據、分析延伸至 2024、製作圖表、文獻回顧、撰寫論文、推送 GitHub
2. **準確率驗證**：29/30 加州縣正確編碼，選舉數據相關係數 > 0.999
3. **模組化工作流程**：一次一個模組，測試通過才繼續
4. **檢查點機制（🛑）**：每階段需人工審批才能進入下一階段
5. **自我修正能力**：AI 自行執行程式碼並修正錯誤，不依賴用戶回報

## 📝 我的註解與思考
這個案例最有價值的不只是結果，而是 Prompt 的設計：
1. **明確的規則約束**：不可協商的規則讓 AI 行為更可預測
2. **模組化 + 測試驅動**：降低錯誤累積風險
3. **人工檢查點**：平衡自動化與人類監督
4. **誠實原則**：要求 AI 承認不確定性，避免幻覺

這套方法論可以推廣到其他領域的 AI 輔助工作。

## 📋 完整 Prompt 模板（備用）

\`\`\`
關鍵工作流程需求

您必須遵守這些規則。這些規則不可協商。

1.絕不在未經測試證明可行前聲稱某功能有效。 撰寫任何程式碼後，立即撰寫並執行測試。如果無法測試，請明確說明。

2.採用模組化工作方式。 一次完成一個模組。每完成一個模組後，報告所建置的內容，展示測試結果，並等待確認後再繼續。

3.自行反覆修正錯誤。 不要依賴使用者回報錯誤。執行程式碼，觀察輸出，並在呈現結果前修正問題。

4.對未知事項保持明確。 如果不確定某件事，請直接說明。不要猜測。

5.使用 python3 和 pip3。 所有指令請務必使用 python3（非 python）及 pip3（非 pip）。

專案說明

請閱讀此目錄中的 INSTRUCTIONS.md。裡面包含了複製及擴展一篇已發表的政治科學論文（關於郵寄投票）的詳細指引。

你的任務：依階段逐步執行指示，從第 0 階段（專案設定）開始。

關鍵規則：在每個標記為🛑的檢查點，您必須：

1.總結您已完成的內容
2.提出關鍵成果以供審查
3.列出任何問題或疑慮
4.停止並等待我明確批准後，方可進入下一階段

請勿跳過檢查點。未經我批准，請勿超過 🛑 。

立即開始第 0 階段。
