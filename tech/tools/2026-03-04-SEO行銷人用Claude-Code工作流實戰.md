# SEO 行銷人用 Claude Code 工作流實戰

## 📊 元資訊
- **難度**：⭐⭐⭐
- **來源類型**：部落格文章
- **作者**：Hiba Fathima（Firecrawl SEO 主管）
- **筆記時間**：2026-03-04 08:49

## 📌 摘要
Firecrawl 的 SEO 主管 Hiba Fathima 分享她如何用 Claude Code（包含 Desktop 版本）處理大部分行銷工作。核心觀點是：這不能取代工程師，但能消除「等別人排程」的問題，讓行銷人自己快速上線各種小工具和自動化流程。

## 🏷️ 標籤分類
- **大分類**：tech
- **小分類**：tools

## 🔑 關鍵要點

### Claude Code 的核心價值
1. **消除依賴等待**：以前活動頁面流程「設計師→工程師→驗收→修改」，現在想清楚就能直接上線
2. **專案級存取**：Claude Code 住在專案裡，能讀取整個目錄、跨檔案修改、執行指令
3. **非技術人員友善**：可用 Desktop App，拖拉截圖、貼圖片，不需面對終端機

### 8 大實戰應用

| 應用 | 說明 | 效益 |
|------|------|------|
| 客製化追蹤工具 | 監測 Reddit/HN/Twitter 上的品牌提及 | 每週自動收到討論報告，找內容靈感 |
| 自動更新過時文章 | Ahrefs 流量數據 → Slack 通知 → Claude 優化 | 省下大量手動更新時間 |
| 內連優化工具 | 基於語意（非關鍵字）建議內部連結 | 取代 $30/月 SaaS，效果更好 |
| 競品分析研究 | 整合 Firecrawl 抓取競品數據 | 自動生成比較表 |
| FAQ 生成 | 用 /agent 找目標受眾真實問題 | 回答真實需求而非瞎猜 |
| 技術雜務自動化 | 圖片優化、網站健康度審核 | WebP 轉換、Lazy loading 等 |
| AI 能見度追蹤 | 分析 AI 爬蟲（ChatGPT-User）爬了哪些頁 | AEO 優化依據 |
| 研究筆記管理 | 本機文字檔存素材，寫作時自動引用 | 像有記憶力超強的研究助理 |

### 重要提醒
- **人要在流程裡**：面向用戶的改動務必請工程師確認
- **不要自動發布**：每項改動親自審閱，AI 生成內容通常普通且不符品牌語氣
- **需要學點基礎技術**：但不需要是工程師

## 🧠 概念連結
- **Firecrawl**：為 AI 應用提供高品質網頁數據擷取 API，將混亂網頁轉為乾淨 Markdown/JSON
- **AEO（Answer Engine Optimization）**：讓內容更容易被 ChatGPT、Perplexity 等 AI 工具引用的優化策略
- **Skills**：Claude Code 的工作 SOP，把重複流程文件化後可重複使用

## 💡 與我的連結
這篇分享的工作流與我目前的 Happy Coder + 知識庫系統有很多相似之處：
- 我也用 Claude Code 打造客製化工具（TG Bot、監控系統）
- 內連優化和 FAQ 生成的思路可以應用到知識庫文章
- AEO 追蹤是新概念，值得研究如何套用

## 📝 我的註解與思考

### 對非工程師的啟發
這篇最有價值的觀點是：行銷人不需要成為工程師，但可以利用 AI 工具消除對工程師排程的依賴。這改變了「誰能做什麼」的邊界。

### Skills 系統的重要性
她特別提到 `hiba-writing-style` 這個 Skill 來維持寫作風格一致性。這提醒我：
- 應該把知識庫筆記格式做成 Skill
- 常用的分析流程也可以 Skill 化

### 人機協作的正確姿勢
她強調「不要自動發布」「人要在流程裡」—— 這是成熟的 AI 使用觀。AI 處理苦差事，人負責決策和品質把關。
