# Claude Code Skills 推薦清單（三層架構）

## 📊 元資訊
- **難度**：⭐⭐
- **來源類型**：文章 / 資源清單
- **作者**：BrowserAct
- **筆記時間**：2026-03-06 11:04

## 📌 摘要
整理 Claude Code Skills 的三層推薦架構，從官方必裝的文件處理 Skills，到進階實用工具，最後是最具影響力的 Skill Creator 和 Superpowers。這份清單幫助使用者系統化地建立 Claude Code 的能力擴展。

## 🏷️ 標籤分類
- **大分類**：tech
- **小分類**：tools

## 🔑 關鍵要點
1. **Layer 1（必裝）**：Anthropic 官方四件套 — PDF、DOCX、PPTX、XLSX 文件處理
2. **Layer 2（進階）**：前端設計、SEO、行銷、Obsidian 整合等專業領域 Skills
3. **Layer 3（核心）**：Skill Creator（自建 Skill）+ Superpowers（需求分析優先）
4. 作者特別推薦 Skill Creator，稱之為「改變一切」的關鍵工具

## 💬 金句摘錄
> "這兩個被作者稱為『改變一切』的 Skills"
> — BrowserAct

## 🧠 概念連結
- **Claude Code CLI**：Skills 是 Claude Code 的擴展機制
- **MCP (Model Context Protocol)**：Skills 與 MCP 都是擴展 Claude 能力的方式
- **Prompt Engineering**：Skill Creator 本質上是結構化的 prompt 管理

## 💡 與我的連結
- 我已經在使用 Claude Code，這些 Skills 可以大幅提升效率
- Skill Creator 可以用來建立針對知識庫管理的客製化 Skill
- Obsidian Skills 可能對知識庫整合有幫助

## ✅ 行動項目
- [ ] 安裝 Layer 1 四件套（PDF、DOCX、PPTX、XLSX）
- [ ] 研究 Skill Creator 如何建立自訂 Skill
- [ ] 評估 Superpowers 是否適合目前工作流程
- [ ] 測試 Obsidian Skills 能否整合到知識庫管理

## 📝 我的註解與思考
Skills 的三層架構很有參考價值：
- Layer 1 是基礎能力，處理日常文件
- Layer 2 是專業垂直領域擴展
- Layer 3 是元能力（meta-skill）— 建立 Skill 的 Skill

Skill Creator 的概念特別重要：與其安裝別人的 Skill，不如學會自己建立。這符合「授人以漁」的原則。
